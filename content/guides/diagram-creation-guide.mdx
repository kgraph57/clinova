# 【ツール別】医学図解作成ガイド：Nanobanana活用

## はじめに

医学教育や臨床現場では、複雑な病態生理、診断フロー、治療アルゴリズムなどを視覚的に説明する図解が不可欠です。
しかし、高品質な医学図解を作成するには、専門的なデザインスキルや時間が必要で、多くの医療従事者にとって大きな障壁となっています。

このガイドでは、NotebookLMのNanobananaを活用して、医学図解を効率的に作成する方法を紹介します。
AIを活用することで、専門的なデザインスキルがなくても、わかりやすく正確な医学図解を作成できるようになります。

![図解作成ワークフロー](/images/guides/diagram-creation-guide/workflow.png)

---

## なぜNanobananaが必要なのか？

### 医学図解作成の課題

従来の図解作成には以下の課題があります：

- **時間がかかる**: 専門的な図解ソフトウェアの習得に時間が必要
- **デザインスキルが必要**: 視覚的にわかりやすい図解を作成するにはデザインの知識が必要
- **正確性の確保が難しい**: 医学的内容を正確に図解化するには専門知識が必要
- **更新が大変**: 新しい知見が得られた際に図解を更新するのが困難

### Nanobananaの利点

NotebookLMのNanobananaを活用することで、以下の利点があります：

- **AIによる自動生成**: テキストから自動的に図解を生成
- **医学的内容に対応**: 病態生理、診断フロー、治療アルゴリズムなどに対応
- **視覚的にわかりやすい**: AIが最適な視覚化方法を提案
- **簡単に更新可能**: テキストを修正するだけで図解を更新

# 基本編 - ステップ1: Nanobananaの基本操作

## はじめに

このステップでは、NotebookLMのNanobananaの基本的な使い方を学びます。
Nanobananaは、テキストから自動的に図解を生成するAIツールです。

---

## Nanobananaとは？

Nanobananaは、NotebookLMに統合されたAI画像生成ツールです。
医学的な内容をテキストで説明すると、それを視覚的な図解に変換してくれます。

### 主な特徴

- **テキストから図解生成**: 医学的内容をテキストで入力すると、自動的に図解を生成
- **医学的内容に対応**: 病態生理、診断フロー、治療アルゴリズムなどに対応
- **高品質な出力**: 医学教育やプレゼンテーションに適した品質の図解を生成

---

## 基本的な使い方

### 1. NotebookLMにアクセス

まず、NotebookLMのウェブサイトにアクセスし、ログインします。

### 2. 新しいノートブックを作成

「新しいノートブック」をクリックして、新しいノートブックを作成します。

### 3. Nanobananaを起動

ノートブック内で、Nanobananaアイコンをクリックして起動します。

### 4. テキストを入力

図解化したい医学的内容をテキストで入力します。

例：
```
心筋梗塞の病態生理：
1. 冠動脈の閉塞により心筋への血流が途絶
2. 心筋細胞の虚血と壊死
3. 心機能の低下
```

### 5. 図解を生成

「生成」ボタンをクリックすると、AIが自動的に図解を生成します。

---

## プロンプトのコツ

効果的な図解を生成するためのプロンプトのコツ：

### 明確な構造を指定

```
以下の内容を図解化してください：
- タイトル: 心筋梗塞の病態生理
- 主要な要素: 冠動脈、心筋、血流
- プロセス: 閉塞→虚血→壊死→機能低下
```

### 視覚的な要素を指定

```
以下の要素を含む図解を作成してください：
- 矢印でプロセスの流れを示す
- 色分けで正常と異常を区別
- ラベルで各要素を明示
```

# 基本編 - ステップ2: 病態生理の図解化

## はじめに

このステップでは、病態生理を図解化する方法を学びます。
病態生理は複雑なメカニズムを視覚的に説明する必要があるため、適切なプロンプト設計が重要です。

---

## 病態生理図解の基本構造

病態生理図解には以下の要素が含まれることが多いです：

- **正常状態**: 正常な生理的プロセス
- **異常状態**: 疾患による変化
- **プロセスの流れ**: 時系列での変化
- **関連要素**: 関連する臓器、細胞、分子など

---

## 効果的なプロンプト例

### 例1: 心筋梗塞の病態生理

```
以下の心筋梗塞の病態生理を図解化してください：

【正常状態】
- 冠動脈から心筋への正常な血流
- 心筋細胞の正常な機能

【異常状態】
- 冠動脈の閉塞（血栓、プラーク）
- 心筋への血流途絶
- 心筋細胞の虚血と壊死
- 心機能の低下

【プロセスの流れ】
1. 冠動脈の狭窄・閉塞
2. 心筋への血流減少
3. 心筋細胞の虚血
4. 心筋細胞の壊死
5. 心機能の低下

【視覚的な要素】
- 正常な状態は青色で表示
- 異常な状態は赤色で表示
- 矢印でプロセスの流れを示す
- 各段階にラベルを付ける
```

### 例2: 糖尿病の病態生理

```
以下の2型糖尿病の病態生理を図解化してください：

【主要な要素】
1. インスリン抵抗性
2. 膵臓のβ細胞機能低下
3. 高血糖状態

【プロセスの流れ】
1. インスリン抵抗性の発生
2. インスリン分泌の増加
3. β細胞の疲弊
4. インスリン分泌の低下
5. 高血糖の持続

【視覚的な要素】
- インスリンを黄色で表示
- 血糖を赤色で表示
- プロセスの流れを矢印で示す
- 各臓器（膵臓、肝臓、筋肉）を明確に区別
```

---

## プロンプトのポイント

### 1. 構造を明確にする

病態生理の各段階を明確に分けて記述します。

### 2. 視覚的な要素を指定

色、矢印、ラベルなどの視覚的な要素を指定します。

### 3. 関連要素を含める

関連する臓器、細胞、分子などを含めます。

### 4. 時系列を明確にする

プロセスの流れを時系列で明確にします。

# 実践編 - ステップ1: 診断フローの図解化

## はじめに

このステップでは、診断フローを図解化する方法を学びます。
診断フローは、臨床現場でよく使用される図解で、診断プロセスを段階的に表示します。

---

## 診断フロー図の基本構造

診断フロー図には以下の要素が含まれます：

- **開始点**: 症状や所見
- **判断点**: 診断の分岐点（Yes/No）
- **診断**: 最終的な診断
- **治療**: 治療方針

---

## 効果的なプロンプト例

### 例1: 胸痛の診断フロー

```
以下の胸痛の診断フローを図解化してください：

【開始】
患者の胸痛

【判断1】
急性冠症候群の可能性？
- Yes → 心電図、心筋マーカー検査
- No → 次の判断へ

【判断2】
肺塞栓の可能性？
- Yes → D-ダイマー、CTPA
- No → 次の判断へ

【判断3】
大動脈解離の可能性？
- Yes → 胸部CT
- No → 次の判断へ

【判断4】
その他の原因？
- 消化器疾患、筋骨格系疾患など

【視覚的な要素】
- 判断点をダイヤモンド型で表示
- 診断を四角形で表示
- 矢印でフローの流れを示す
- Yes/Noを明確にラベル付け
```

### 例2: 発熱の診断フロー

```
以下の発熱の診断フローを図解化してください：

【開始】
患者の発熱

【判断1】
感染症の可能性？
- Yes → 血液培養、画像検査
- No → 次の判断へ

【判断2】
自己免疫疾患の可能性？
- Yes → 自己抗体検査
- No → 次の判断へ

【判断3】
悪性腫瘍の可能性？
- Yes → 画像検査、生検
- No → その他の原因を検討

【視覚的な要素】
- フローチャート形式で表示
- 各判断点を明確に区別
- 診断への経路を矢印で示す
```

---

## プロンプトのポイント

### 1. 判断点を明確にする

各判断点で何を評価するかを明確にします。

### 2. Yes/Noの分岐を明確にする

各判断点でのYes/Noの分岐を明確にします。

### 3. 診断への経路を明確にする

各診断に至る経路を明確にします。

### 4. 視覚的な構造を指定

フローチャートの形式を指定します。

# 応用編 - ステップ2: 応用テクニックとトラブルシューティング

## はじめに

このステップでは、Nanobananaを使った医学図解作成の応用テクニックと、よくある問題の解決方法を学びます。

---

## 応用テクニック

### 1. 複雑な内容の段階的図解化

複雑な内容は、複数の図解に分けて作成します。

**例：**
```
以下の内容を3つの図解に分けて作成してください：

図解1: 正常な生理的プロセス
図解2: 疾患による変化
図解3: 治療による改善
```

### 2. 既存の図解の改善

生成された図解が期待通りでない場合、具体的な修正指示を追加します。

**例：**
```
以下の図解を改善してください：
- 矢印の色を赤から青に変更
- ラベルのフォントサイズを大きく
- 背景色を白に変更
```

### 3. スタイルの統一

複数の図解を作成する場合、スタイルを統一します。

**例：**
```
以下の図解を作成してください：
- 色スキーム: 青、赤、緑を使用
- フォント: ゴシック体
- 背景: 白
- 矢印: 太い実線
```
