# プレゼン改善フィードバック

## このテクニックとは

カンファ・抄読会・学会発表の前に、自分の発表内容を客観的に見直すことは難しいです。同僚に頼もうとしても、皆忙しく、丁寧なフィードバックをもらえる機会は限られています。AIを使えば、発表内容を貼り付けるだけで「論理的に飛躍している箇所」「スライドで補強すべき点」「質問されそうなポイント」を即座に指摘してもらえます。

このテクニックが特に有効なのは、**発表の役割・文脈を明示する**ことです。「研修医のケースプレゼン」と「学会での口頭発表」ではフィードバックの視点がまったく異なります。「誰が」「誰に向けて」「何の目的で」発表するかをプロンプトに含めることで、的確で実用的なフィードバックが得られます。

また、フィードバックを受けた後に「指摘した問題点を修正したバージョンを書いてください」と続けることで、改善案もまとめて生成できます。

## 基本パターン

```text
あなたは経験豊富な指導医（または学会座長）です。以下の発表内容にフィードバックを行ってください。

【発表の文脈】
- 発表者: [例: 後期研修医2年目]
- 発表形式・場: [例: 院内症例カンファレンス（30分）/ 学会口頭発表（10分）]
- 聴衆: [例: 指導医・研修医・コメディカル混在]
- 発表の目的: [例: 稀な疾患を経験したので症例共有と教育的な議論がしたい]

【発表内容（スライドの文字起こし / アウトライン / 発表原稿）】
[ここに発表内容を貼り付ける]

以下の観点でフィードバックしてください:
1. 全体構成の評価（論理の流れ、時間配分の妥当性）
2. 内容面の問題点（情報の過不足、解釈の妥当性、データの使い方）
3. 想定される質問（上位3件）と理想的な回答のヒント
4. 改善提案（優先度: 必須 / できれば の2段階で）
5. この発表の強みと良かった点

厳しすぎず、建設的なトーンでフィードバックしてください。
```

## 医療での活用例

### シナリオ

発表前に内容をチェックしたい。

### プロンプト例

```text
以下の発表内容にフィードバックをください：

[発表内容]

確認ポイント:
- 論理の一貫性
- データ解釈の妥当性
- 質問されそうなポイント
- 改善提案
```

## いつ使うべきか

- 学会発表前日・週末など、同僚に頼みにくいタイミングでフィードバックが必要なとき
- ケースプレゼンの構成が正しいか、情報の抜け漏れがないか確認したいとき
- 指導医からのフィードバックを受ける前に自己点検として使いたいとき
- 英語発表の原稿を用意し、内容の論理性と英語表現の両方をチェックしたいとき
- 過去の発表録画やスライドを見直し、次の発表への改善点を整理したいとき
