# 忙しい人に読んでもらえる文章術

## このテクニックとは

医療現場の文書は「忙しい人が読む」ものです。指導医への報告、他科への紹介状、カンファレンスの資料——いずれも読み手は限られた時間の中で要点を素早く把握しようとしています。AIに文書を生成させる際に「簡潔さ・読みやすさ」を明示的に指示することで、長い文章ではなく実際に読んでもらえる文書を作れます。

読みやすい医療文書の特徴は「結論先行」「箇条書きの活用」「重要情報の視覚的強調」「不必要な前置きの排除」です。AIはデフォルトでは説明的で冗長になる傾向があるため、「3行以内で」「箇条書き3点で」「結論から先に」のような制約を明示することで、実用的な出力に変わります。

さらに読者に応じて「専門用語のレベル」を調整することも重要です。「指導医向け」と「患者向け」では使える言葉が全く異なります。AIへの指示に「対象読者」と「専門知識レベル」を明記することで、適切な言語レベルの文書が生成されます。

## 基本的な使い方

```text
以下の内容を、忙しい[対象読者]が30秒で要点を掴めるように整理してください。

【内容】
[伝えたい情報]

【制約】
- 最重要ポイントを冒頭に（結論先行）
- 箇条書きで3〜5点以内に絞る
- 一文は50字以内
- 専門用語は使用OK（対象: 医師・看護師）
- 余分な前置き・謝辞は不要
```

患者向け説明文書のテンプレート：

```text
以下の医療情報を患者向けに書き直してください。

【元の内容（医療用語）】
[医師向けの説明や所見]

【書き直しの条件】
- 中学生でも理解できる言葉を使う
- 医療用語は使わない（使う場合は必ず括弧内で説明を付ける）
- 「なぜそうなるのか」を1文で説明する
- 患者が「次に何をすべきか」を明確に書く
- 不安を与える表現は避け、前向きな表現を使う
- 全体を300字以内にまとめる
```

当直医への申し送り（SBAR形式）：

```text
以下の情報をSBAR形式で申し送り文にしてください。30秒で伝えられる簡潔さを最優先にしてください。

S（Situation/状況）: 今何が起きているか、1〜2文で
B（Background/背景）: 関連する重要な既往・経過、2〜3点の箇条書きで
A（Assessment/評価）: 問題の緊急度と最も考えられる状態
R（Recommendation/提案）: 今すぐお願いしたいこと

【情報】
[患者情報・症例情報]
```

## 医療での活用例

### シナリオ

外来診療の合間に複数の患者の検査結果を迅速に共有する必要がある。忙しい医師や看護師が短時間で症例の要点を理解できるよう、簡潔な報告書を作成しなければならない。

### プロンプト例

```text
以下の[患者情報]と[症例情報]をもとに、忙しい医療従事者が短時間で理解できるよう、専門用語を適切に用いて簡潔かつ明瞭な文章で報告書を作成してください。重要な診断、検査結果、治療方針を3つのポイントに絞り、箇条書き形式で示してください。冗長な表現は避け、読みやすさを最優先にしてください。

[患者情報]:
[症例情報]:
```

## いつ使うべきか

- 当直・救急での口頭申し送り原稿を素早く整理したいとき
- 指導医へのプレゼンテーションのサマリーを作成したいとき
- 他科・他院への依頼メールや紹介文を簡潔にまとめたいとき
- 患者・家族向けの説明文書を平易な言葉で作成したいとき
- 研究論文のアブストラクトや抄録を一般向けにリライトしたいとき
