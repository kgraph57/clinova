# 当直サマリー作成

## このテクニックとは

当直明けの疲労困憊な状態で、出来事を時系列で思い出しながら引き継ぎサマリーを書くことは、精神的・身体的に負担の大きい作業です。情報の抜け漏れや優先順位の誤りが生じやすく、日勤帯のチームが重要な情報を把握できないまま業務を始めてしまうリスクがあります。

このテクニックでは、当直中にメモ・箇条書き・走り書きで記録したことをAIに渡し、申し送りに適したサマリー形式に整形させます。重要度順の並び替え、各患者の簡潔なまとめ、継続フォローが必要な項目の明示といった作業を自動化することで、当直明けの最後の仕事を楽にします。

メモは乱雑でも構いません。「ER: 70代男性 腹痛 CT所見異常なし 帰宅」「病棟コール: Aさん SpO2低下 → 酸素増量で改善」のような断片的なメモでも、AIが整理・構造化してくれます。

## 基本パターン

```text
あなたは病院の引き継ぎ担当です。以下の当直中のメモ・出来事リストを、日勤チームへの申し送りサマリーとして整形してください。

【当直中の出来事（メモ・走り書きそのまま）】
[例:
- 23時: ER 68歳男性 胸痛 心電図正常 CK正常 → 様子見で入院
- 2時: 病棟 Bさん(大腸Ca術後) 発熱38.5℃ 血培採取 セフトリアキソン開始
- 4時: Cさん(心不全) SpO2低下 → フロセミド追加 改善
- 朝: ER 84歳女性 転倒 Xp骨折なし 息子に連絡済み 帰宅
]

以下の形式で出力してください:

【優先フォローアップ（要注意患者）】
※継続対応が必要な患者を重要度順に記載

【新規入院患者】
※患者ごとに: 経緯・現在の状態・指示内容

【病棟コール対応済み患者】
※患者ごとに: 問題・対応・現在の状態・追加フォロー必要性

【救急帰宅患者】
※帰宅指示の内容と再来指示

【申し送り事項・未完了タスク】
```

## 医療での活用例

### シナリオ

当直明けで疲れている。サマリーを書く気力がない。

### プロンプト例

```text
以下の当直中の出来事を、申し送り用サマリーにまとめてください。
- 重要度順に記載
- 各患者は30秒で読める長さ
- 継続フォローが必要な項目を明記

[当直中の出来事]
```

## いつ使うべきか

- 当直明けで睡眠不足・疲労状態にある朝の引き継ぎ前
- 当直中に多数の患者対応があり、出来事の整理に時間がかかるとき
- 当直メモが断片的・乱雑で、そのまま読み上げても伝わらない形のとき
- 引き継ぐ相手（日勤チーム）が多く、簡潔で統一されたフォーマットが求められるとき
- 当直報告を電子カルテに入力する際の叩き台として
