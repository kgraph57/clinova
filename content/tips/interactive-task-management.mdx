# 対話型タスク管理

## このテクニックとは

対話型タスク管理は、複雑な医療タスクを一度に処理しようとせず、AIとの対話を通じて段階的に進めていく手法です。最初に大まかな方向性を確認し、フィードバックを受けながら各ステップを洗練させることで、単一のプロンプトでは得られない精度の高い成果物を作成できます。

この手法の核心は「チェックポイントの設置」にあります。AIに次のアクションを実行する前に「ここまでの方針でよいか確認してください」と明示的に求めることで、誤った方向への進行を早期に修正できます。長大な医療文書の作成や、多段階の意思決定プロセスに特に効果的です。

医療現場では、治療計画の立案、退院調整の手順整理、多職種カンファレンスの準備など、複数のステップと意思決定が連鎖するタスクに日常的に直面します。対話型タスク管理を活用することで、AIを「一度きりの回答機械」ではなく「作業を共に進めるパートナー」として機能させられます。

## 基本的な使い方

```text
以下のタスクを段階的に進めてください。各ステップ完了後に確認を求めてから次に進んでください。

【タスク】
[実行したいタスクの概要]

【ステップ1】
[最初に行うこと]
→ 完了したら「ステップ1完了。次に進んでよいですか？」と確認すること

【ステップ2】
[次に行うこと]
→ 完了したら「ステップ2完了。次に進んでよいですか？」と確認すること

【ステップ3】
[最後に行うこと]

各ステップで不明点や追加情報が必要な場合は、進める前に質問してください。
```

複数の患者を同時に管理する場合のテンプレート：

```text
以下の患者リストについて、優先度評価→情報収集→対応方針の3ステップで管理してください。

【患者リスト】
[患者A]: [主訴・状態]
[患者B]: [主訴・状態]
[患者C]: [主訴・状態]

ステップ1: まず重症度・緊急度でトリアージし、対応優先順位をつけてください。優先順位が確定したら次のステップに進みます。
ステップ2: 最優先患者から順に、必要な追加情報と確認事項をリストアップしてください。
ステップ3: 各患者の初期対応方針を提示してください。

準備できたら「ステップ1から開始します」と宣言してから始めてください。
```

## 医療での活用例

### シナリオ

多職種チームが複雑な患者の治療計画を立案するため、AIと対話しながら段階的に診断と治療方針を整理している。各ステップで情報を確認しつつ、適切な検査や処方を決定する必要がある場面。

### プロンプト例

```text
退院支援カンファレンスの準備を段階的に進めてください。

【患者情報】
78歳男性、脳梗塞後遺症（左片麻痺）、認知症（MMSE 18点）、
独居、ADL：食事一部介助・移動車椅子・排泄一部介助

ステップ1: 退院先の選択肢（自宅・施設・転院）を、医学的観点と社会的観点から評価してください。評価が終わったら確認してください。

ステップ2: 最適と判断された退院先に向けて、今週中に調整すべき事項をリストアップしてください。

ステップ3: カンファレンスで各職種（医師・看護師・PT・MSW）が確認すべき事項を整理してください。

各ステップ完了後に「完了しました。次のステップに進んでよいですか？」と確認を求めること。
```

## いつ使うべきか

- 治療計画立案など複数の意思決定が連続するタスクを整理したいとき
- 退院支援・転科調整など多職種が関わる複雑な調整作業の準備
- カルテ記録・紹介状・退院サマリーなど長文書類を段階的に作成するとき
- 研修医や後期研修医の教育ケースを構造化して提示したいとき
- カンファレンス前に症例の論点を順序立てて整理したいとき
