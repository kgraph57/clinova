# 事実確認

## このテクニックとは

AIは流暢に文章を生成するが、自信ありげに誤った情報を提示するハルシネーションが起きうる。医療の文脈では、薬剤の用量・禁忌・臨床試験の結果・診断基準の数値などの「事実」が1つ間違うだけで患者安全に影響する。「事実確認モード」で依頼することで、AIが情報の確実性を自己評価しながら回答するよう促せる。

このテクニックの核心は「確信度の明示」を求めることだ。「この情報は確実なエビデンスに基づくか、それとも推論や一般論か」をAIに区別させることで、医師がどこを追加確認すべきかが明確になる。特に「最新の情報」「稀な疾患」「マイナーな薬剤」についての記述は、AIの訓練データに含まれていない可能性があり、事実確認の指定が重要になる。

AIの出力をそのまま信じるのではなく、「どの部分を自分でダブルチェックすべきか」を可視化するための道具として活用するのが正しい使い方だ。

## 基本パターン

```text
以下の[医療情報/診断結果/薬剤情報]について、事実確認を行いながら回答してください。

【確認してほしい情報】
[確認したい内容・AIの出力・症例情報など]

回答の際は以下のルールを守ってください：
1. 確実なエビデンス（ガイドライン・主要RCT）に基づく情報には「確実」と付記する
2. 一般的知識や推論に基づく場合は「一般的には」「推論」と明示する
3. 知識が古い・更新された可能性がある場合は「要確認」と付記する
4. 数値（用量・基準値・試験結果）は特に注意して、不確実な場合は範囲や「要確認」を示す
5. ハルシネーションが起きやすい項目（臨床試験の細かい数値・稀な疾患の統計）は特に注意する

最後に「この回答で特に現物確認を推奨する箇所」をリストアップしてください。
```

## 医療での活用例

### シナリオ

救急外来で複数の臨床データが提示され、迅速かつ正確な診断が求められている。医療スタッフはAIからの診断サポート情報の信頼性を確認し、誤診を防止したいと考えている。

### プロンプト例

```text
以下の[症例情報]と[患者情報]をもとに、診断および推奨される治療方針についてステップバイステップで論理的に説明してください。各ステップで用いた医学的根拠やエビデンスを明示し、情報の正確性と信頼性を確認してください。特に重要な検査結果や症状については、最新の臨床ガイドラインに基づく解説を含めてください。

【症例情報】
[症例情報]

【患者情報】
[患者情報]

以上を踏まえて、事実確認を重視した診断プロセスを詳述してください。
```

## いつ使うべきか

- AIが提示した薬剤の用量・禁忌・相互作用を処方前に確認したいとき
- 稀な疾患・マイナーな薬剤について情報を収集し、正確性を担保したいとき
- AIが生成した内容を患者や家族へ提供する前に、医学的正確性を保証したいとき
- 研修医がAIを使って調べた情報を「どこまで信頼してよいか」を指導するとき
- 論文・報告書にAIの出力を参考にした情報を記載する前に、事実確認を行うとき
