# 反復的改善

## このテクニックとは

反復的改善は、AIが最初に出力した内容を出発点とし、フィードバックを与えながら繰り返し改良していく手法です。一度の完璧な出力を目指すのではなく、「粗い → 良い → 最良」というサイクルを意図的に回すことで、単発のプロンプトでは到達しにくい高品質な成果物を得られます。

このアプローチが有効な理由は、最初の出力が「たたき台」として機能するからです。白紙から完璧なものを作るより、具体的な草案に対して「ここが違う」「この視点が足りない」とフィードバックする方が、人間にとっても思考しやすく、AIにとっても修正の方向が明確になります。

医療文書の作成（紹介状・退院サマリー・説明文書）や治療計画のブラッシュアップに特に効果的です。最初の草稿は5分でも、複数回のフィードバックで臨床的に十分な精度に仕上げることができます。

## 基本的な使い方

```text
以下の情報をもとに、[作成物]の初稿を作成してください。

【情報】
[必要な情報を記載]

【初稿の条件】
- [形式・分量・対象読者など]

初稿を確認後、改善点をフィードバックします。
```

フィードバックの返し方（第2ターン以降）：

```text
初稿を確認しました。以下の点を修正してください：

1. [修正点1]: [具体的にどう変えてほしいか]
2. [修正点2]: [具体的にどう変えてほしいか]
3. [修正点3]: [具体的にどう変えてほしいか]

上記を反映した改訂版を作成してください。
```

段階的改善を最初から設計する場合：

```text
以下の症例情報をもとに、3段階で診断案を改善してください。

【症例】
[症例情報]

第1段階: 手持ち情報だけで考えられる鑑別診断トップ3を提示
第2段階: 各鑑別の問題点・不整合を自己指摘し、追加すべき検査を提案
第3段階: 第2段階の考察を踏まえた修正診断案と治療方針を提示

各段階で改善した理由を明記してください。
```

## 医療での活用例

### シナリオ

救急外来で複雑な症例の診断が求められている。医師は限られた時間内に患者の症状、検査結果、既往歴をもとに最適な診断と治療計画を立てる必要がある。AIを利用して段階的に診断の精度を高めたい状況。

### プロンプト例

```text
以下の[症例情報]と[患者情報]を基に、初回診断案を提示してください。その後、提示された診断案の問題点や不足情報を指摘し、追加で必要な検査や鑑別診断の候補を提案してください。これを踏まえ、改善案を反映した修正診断案を段階的に示してください。ステップバイステップで考察し、各段階での変更理由を明確にしてください。

【症例情報】
60歳男性、突然発症の激しい頭痛（VAS 9/10）、嘔吐2回
既往: 高血圧（内服中）
バイタル: BP 180/110、HR 88、SpO2 99%、体温 37.1℃
神経学的所見: 項部硬直あり、意識清明、局所神経症状なし
```

## いつ使うべきか

- 紹介状・退院サマリー・患者説明文書など医療文書の草稿作成と改善
- 診断が難しい症例で、初期仮説を精緻化していくとき
- プレゼンテーション用スライドや学会発表の原稿ブラッシュアップ
- 臨床研究のプロトコルや倫理審査書類の内容を段階的に改善したいとき
- 研修医が書いたカルテ記録を指導医がAIを使って改善例を示すとき
