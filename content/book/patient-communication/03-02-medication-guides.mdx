## 服薬アドヒアランスの現実

処方された薬を正しく飲み続けることは、患者にとって想像以上に難しい。

2024年の WHO Global Report on Medication Adherence によると、慢性疾患患者の服薬アドヒアランスは平均50%にとどまる。日本国内のデータも同様である。

- **降圧薬**: 処方開始1年後の継続率は**62%**
- **糖尿病薬**: 6ヶ月後のアドヒアランス良好率は**54%**
- **スタチン**: 2年後の継続率は**45%**
- **抗うつ薬**: 3ヶ月以内の自己中断率は**38%**

<Callout type="warning" title="服薬ノンアドヒアランスの経済的影響">
服薬ノンアドヒアランスによる医療費の損失は、日本国内で年間推定1.5兆円とされる。入院の10〜25%が服薬不良に起因するという報告もある。適切な服薬指導資料の提供は、患者の健康と医療経済の両方に貢献する。
</Callout>

### ノンアドヒアランスの原因分析

患者が薬を飲まない理由は多岐にわたる。AIで服薬指導資料を作成する際は、これらの原因を事前に予測し、対応することが重要である。

| 原因カテゴリ | 具体例 | 割合 |
|------------|--------|------|
| 意図的中断 | 副作用が嫌、必要性を感じない | 35% |
| 非意図的中断 | 飲み忘れ、外出時に持参しない | 40% |
| 経済的理由 | 薬代が高い、通院の負担 | 12% |
| 理解不足 | 飲み方がわからない、説明を忘れた | 13% |

## AI活用の服薬指導資料設計

### 基本テンプレートの構成

効果的な服薬指導資料は、以下の要素を含む。

<StepFlow>
<StepFlowStep number={1} title="薬の名前と外観">
薬の名前（商品名と一般名）、色、形、大きさ。「白い丸い錠剤」のような視覚的な記述を含める。可能であれば薬の写真やイラストを添付する。
</StepFlowStep>
<StepFlowStep number={2} title="なぜこの薬を飲むのか">
「血圧を下げるため」ではなく「脳卒中や心臓病を防ぐため」のように、患者にとっての利益を明示する。薬の必要性を理解することがアドヒアランスの最大の動機づけとなる。
</StepFlowStep>
<StepFlowStep number={3} title="いつ・どのように飲むか">
「毎食後」ではなく「朝食を食べ終わったらすぐに」のように、具体的なタイミングと方法を指示する。食事との関係、水の量、噛んでよいかなどの詳細を含める。
</StepFlowStep>
<StepFlowStep number={4} title="飲み忘れたときの対処">
「次の分から飲んでください」のような曖昧な指示ではなく、時間の目安を明示する。「気づいたのが次の服用時刻の4時間以上前なら、気づいた時点で1回分を飲む。4時間以内なら飛ばして次の時間に飲む」など。
</StepFlowStep>
<StepFlowStep number={5} title="注意すべき副作用と対処法">
よくある副作用（頻度5%以上）と、まれだが重大な副作用に分けて記載する。副作用が出たときの行動指示（様子を見る/次の受診で相談/すぐに受診）を明示する。
</StepFlowStep>
</StepFlow>

### ポリファーマシー患者の統合資料

複数の薬を服用する患者には、薬剤ごとの資料ではなく、統合的な服薬スケジュール資料が効果的である。

<PromptTemplate title="ポリファーマシー患者の統合服薬スケジュール">
以下の処方内容を、患者が毎日参照できる統合服薬スケジュールとして作成してください。

**処方内容**:
1. アムロジピン5mg 1錠 朝食後
2. カンデサルタン8mg 1錠 朝食後
3. メトホルミン500mg 2錠 朝夕食後
4. ロスバスタチン5mg 1錠 夕食後
5. アスピリン100mg 1錠 朝食後
6. ランソプラゾール15mg 1錠 朝食前

**出力形式**:

1. **タイムライン形式のスケジュール表**
   ```
   朝起きたら → 朝食前の薬 → 朝食 → 朝食後の薬 → ... → 夕食 → 夕食後の薬
   ```

2. **薬の一覧表**（各薬に番号を振る）
   - 番号、名前、色と形、何のための薬か、飲むタイミング

3. **曜日別チェック表**（1週間分）
   - 朝食前 □ ランソプラゾール
   - 朝食後 □ アムロジピン □ カンデサルタン □ メトホルミン □ アスピリン
   - 夕食後 □ メトホルミン □ ロスバスタチン

4. **飲み忘れ対処一覧**
   - 薬ごとに「気づいた時点で飲む」or「次まで待つ」の判定基準

5. **旅行・外出時のチェックリスト**

## 副作用説明の最適化

### 恐怖と安心のバランス

副作用の説明は、過不足なく行う必要がある。説明しすぎると服薬拒否（ノセボ効果）を招き、説明不足は信頼の喪失につながる。

<PromptTemplate title="副作用説明の最適化プロンプト">
以下の薬剤の副作用情報を、患者が安心して服薬を始められる形式で作成してください。

**薬剤**: スタチン（ロスバスタチン5mg）

**副作用の記載ルール**:

1. **よくある（10人中1人以上）**:
   - 具体的な症状と対処法
   - 「多くの場合、飲み続けるうちに軽くなります」

2. **ときどき（100人中1〜10人）**:
   - 具体的な症状
   - 「気になったら次の受診で相談してください」

3. **まれだが重要（1,000人中1〜10人）**:
   - 症状と「この症状が出たらすぐに連絡してください」
   - 連絡先を明記

4. **ノセボ効果への対策**:
   - 「薬の説明書には多くの副作用が書いてありますが、ほとんどの方は問題なく服用しています」
   - 数字で安心感を提供（「100人中95人は副作用なく飲めています」）

**出力形式**: 信号機方式
- 🟢 緑: 心配いりません（よくある軽い症状）
- 🟡 黄: 次の受診で相談（気になる症状）
- 🔴 赤: すぐに連絡（重大な症状）
</PromptTemplate>

<Callout type="info" title="ノセボ効果に注意">
副作用を詳細に説明しすぎると、実際には薬と無関係な症状まで「副作用だ」と感じてしまうノセボ効果が生じる。2024年のBMJ研究では、副作用リストを詳細に読んだ群は、要約版を読んだ群より副作用の訴えが23%多かった。情報の「適切な粒度」が重要である。
</Callout>
