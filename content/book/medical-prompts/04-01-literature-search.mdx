## 文献検索へのAI活用

文献検索は医療者の日常業務の一部ですが、効率的な検索戦略の設計は意外と難しい作業です。AIは検索式の構築、PIKOの設定、文献の批判的吟味の構造化に活用できます。

<Warning>
LLMは架空の文献（ハルシネーション）を生成する可能性があります。AIが挙げた論文は、必ずPubMed等のデータベースで実在を確認してください。特にDOI番号や著者名が正確かどうかの検証が必要です。
</Warning>

## PubMed 検索戦略

<PromptTemplate title="PubMed検索戦略設計プロンプト">
あなたは医学図書館司書で、PubMed検索のエキスパートです。

以下のPICOに基づく包括的なPubMed検索戦略を設計してください。

【PICO】
P: [Patient]
I: [Intervention]
C: [Comparison]
O: [Outcome]

【検索戦略の出力】
■ 検索ブロック
Block 1（Population）:
  MeSH: [terms]
  Text words: [terms]
  組み合わせ: #1 OR #2 OR #3...

Block 2（Intervention）:
  MeSH: [terms]
  Text words: [terms]
  組み合わせ: #4 OR #5 OR #6...

Block 3（Outcome）（必要に応じて）:
  MeSH: [terms]
  Text words: [terms]

■ 最終検索式
(Block 1) AND (Block 2) [AND (Block 3)]

■ フィルター推奨
- Study design: [RCT/SR/Meta-analysis等]
- Language: [English/Japanese]
- Date: [期間]
- Human subjects

■ 推定ヒット数の予想
- フィルターなし: [推定]
- フィルターあり: [推定]

■ 補足検索
- Cochrane Library の検索
- 医中誌の検索式（日本語文献用）
- ハンドサーチが推奨されるジャーナル

【制約】
- MeSH termsは最新のMeSH databaseに準拠
- 感度を重視した検索（系統的レビュー向け）と、特異度を重視した検索（臨床疑問解決向け）の2パターン提示
- 検索式は再現可能で、検索日を記録する前提
</PromptTemplate>

## 文献の批判的吟味

<PromptTemplate title="RCTの批判的吟味プロンプト">
あなたはEBMの専門家で、論文の批判的吟味に精通しています。

以下のRCTの論文情報について、批判的吟味を行ってください。

【論文情報】
- タイトル: [タイトル]
- ジャーナル: [ジャーナル名]
- 対象: [対象患者]
- 介入: [介入内容]
- 主要アウトカム: [アウトカム]
- 主な結果: [結果のサマリー]
- サンプルサイズ: [N]
- フォロー期間: [期間]

【批判的吟味チェックリスト（CONSORT準拠）】
□ 研究デザインの妥当性
  - ランダム化の方法は適切か
  - 割り付けの隠蔽は行われたか
  - 盲検化は適切か（誰が盲検化されたか）

□ 内的妥当性
  - ITT解析が行われたか
  - 脱落率は許容範囲か（&lt;20%）
  - ベースラインの群間差はないか
  - 交絡因子は適切に調整されたか

□ 結果の解釈
  - 統計的有意性と臨床的意義は区別されているか
  - 信頼区間は示されているか
  - NNT/NNHは算出可能か
  - サブグループ解析は事前に計画されたか

□ 外的妥当性
  - 自分の患者に適用可能か
  - 日本人データへの一般化可能性

【出力形式】
■ エビデンスレベルの評価: [GRADE]
■ 論文の強み: [3点]
■ 論文の限界: [3点]
■ 臨床適用の判断: [適用可能/条件付き/適用困難]
■ 「この論文を一言でまとめると」: [1文]
</PromptTemplate>

## 系統的レビューの支援

<PromptTemplate title="系統的レビュー計画プロンプト">

【研究テーマ】
[テーマの記述]

【計画の出力】

4. スクリーニングプロセス
   - タイトル/アブストラクトスクリーニング
   - フルテキストスクリーニング
   - 2名以上の独立した評価

5. データ抽出フォーム
   | 項目 | 詳細 |
   |------|------|
   | 著者/年 | |
   | 研究デザイン | |
   | 対象 | |
   | 介入 | |
   | アウトカム | |
   | 結果 | |
   | バイアスリスク | |

6. バイアスリスク評価
   - RoB2（RCTの場合）/ ROBINS-I（観察研究の場合）

<Callout>
文献検索プロンプトの注意点：
- **ハルシネーション**: AIは架空の論文を生成する可能性あり、必ず実在を確認
- **最新文献**: AIの学習データには直近の論文が含まれない場合あり
- **検索式の検証**: AI生成の検索式はPubMedで実際に検索して確認
- **批判的吟味**: AIの評価は参考にし、自分の目でも論文を読む
</Callout>

<KeyTakeaway>
文献検索プロンプトの最大の活用法は、PICOの構造化と検索戦略の設計です。AIは検索式の構築、批判的吟味のフレームワーク適用、論文の要約に有用ですが、AIが挙げる文献の実在確認は必ず行ってください。系統的レビューではAIは計画段階の補助に留め、実際のスクリーニングやデータ抽出は人間が行う必要があります。
</KeyTakeaway>
