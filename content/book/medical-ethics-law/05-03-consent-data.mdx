**臨床の場で収集された患者データを、AI開発という当初の目的と異なる用途に「二次利用」する場合、どのような同意が必要か。この問題は、個人情報保護法・倫理指針・国際規範が交錯する複雑な領域であり、医療AI開発の根幹に関わる。**

---

## データの二次利用とは

### 定義

データの二次利用（secondary use）とは、当初の収集目的（一次利用：臨床診療）とは異なる目的（二次利用：研究、AI開発等）にデータを使用することである。

### 医療AIにおける二次利用の具体例

- 診療で取得したCT画像をAI画像診断モデルの学習に使用する
- 電子カルテのデータをAI予測モデルの開発に使用する
- 血液検査の結果を疾患リスク予測AIの検証に使用する
- 診療録のテキストデータをLLMのファインチューニングに使用する

<Callout type="info" title="一次利用と二次利用の境界">
同じ患者のケアの質向上のためにAIを使用する場合（例：自院のデータで学習したAIを自院の患者に使用する場合）は、一次利用の延長として解釈できる余地がある。しかし、他の患者や他の医療機関のために使用する場合は、明確に二次利用である。
</Callout>

## 日本の法的枠組み

### 個人情報保護法

個人情報保護法は、利用目的の変更を「変更前の利用目的と関連性を有すると合理的に認められる範囲」に限定している（第17条第2項）。診療目的で収集されたデータをAI開発に使用することが、この「合理的に認められる範囲」に含まれるかどうかは、個別の判断が必要である。

**原則**：利用目的の範囲を超える場合は、あらかじめ本人の同意が必要（第18条第1項）。

**例外**（第18条第3項）：
1. 法令に基づく場合
2. 人の生命、身体または財産の保護に必要であり、本人の同意取得が困難な場合
3. 公衆衛生の向上または児童の健全な育成の推進に特に必要であり、本人の同意取得が困難な場合
4. 国の機関等が法令の定める事務を遂行するために協力する場合

### 人を対象とする生命科学・医学系研究に関する倫理指針

「人を対象とする生命科学・医学系研究に関する倫理指針」（令和3年文部科学省・厚生労働省・経済産業省告示第1号）は、医療データを研究目的で二次利用する際の倫理的手続きを定めている。

**原則**：研究対象者からのインフォームドコンセント（IC）の取得が必要。

**IC手続きの簡略化が認められる場合**：
- 既存試料・情報の利用であること
- 研究対象者に対するリスクが最小限であること
- 研究の実施が困難になること
- 適切な情報公開（オプトアウト）を行うこと
- 倫理審査委員会の承認を得ること

## 同意のモデル

### 従来型インフォームドコンセント（Specific Consent）

特定の研究に対する具体的な同意。同意の対象（何のために、どのようにデータが使用されるか）が明確であり、患者の自律性が最も尊重される。

**課題**：AI開発の場合、将来どのようなAIが開発されるか、どのような分析が行われるかを事前に完全に予測することが困難であるため、具体的な同意の取得が実務上難しい。

### 包括同意（Broad Consent）

将来の研究利用を含む広範な同意。「将来の医学研究にデータを使用することに同意する」という形式。

**利点**：データの長期的な利用が可能。再同意の手続きが不要。
**課題**：患者が同意時に具体的な利用内容を理解できない。自律性の尊重が希薄になる。

### ダイナミックコンセント（Dynamic Consent）

ICT（情報通信技術）を活用し、研究の進行に応じて継続的に同意を取得・更新するモデル。患者がウェブポータルやアプリを通じて、自分のデータの利用状況を確認し、利用の同意・撤回を随時行える。

**利点**：患者の自律性と情報権を継続的に保障できる。AI開発の進行に応じた柔軟な同意管理が可能。
**課題**：システムの構築・維持コスト。デジタルデバイドへの対応。同意疲れ（consent fatigue）のリスク。

### メタコンセント（Meta-Consent）

患者が「どのような種類の同意モデルを希望するか」を事前に選択するモデル。例えば、「低リスクの研究は包括同意でよいが、遺伝情報に関する研究は個別に同意したい」という形式。

## 透明性の確保

### なぜ透明性が必要か

データの二次利用における透明性は、以下の理由から不可欠である。

1. **患者の信頼の基盤**：データがどのように使われているかを患者が知ることで、医療機関への信頼が維持される
2. **自律的判断の前提**：同意の有効性は、十分な情報に基づく判断であることが前提
3. **社会的責任**：公共的な性格を持つ医療データの利用には、社会に対する説明責任がある
4. **法的義務**：個人情報保護法は、利用目的の公表・通知を義務付けている（第21条）

### 透明性確保の実務

**情報公開の内容**：
- データの利用目的（どのようなAIの開発に使用するか）
- データの範囲（どのようなデータが対象か）
- データの管理者と責任者
- データの保管期間と削除方法
- 患者がデータ利用を拒否する方法（オプトアウトの手続き）
- データの共有先（第三者提供がある場合）
- 研究の成果の公表方法

**情報公開の方法**：
- 院内掲示
- ウェブサイトでの公開
- 患者向け説明文書の配布
- 同意取得時の口頭説明
- データ利用状況の年次報告

## 倫理審査委員会（IRB）の役割

### 二次利用における倫理審査

医療データのAI開発への二次利用は、「人を対象とする生命科学・医学系研究に関する倫理指針」に基づく倫理審査委員会（IRB：Institutional Review Board）の審査を受けることが原則として必要である。

### IRBが審査すべき事項

<StepFlow>
<StepFlowStep number={1} title="研究計画の妥当性">
AI開発の目的、方法、期待される成果が科学的・医学的に妥当であるかを審査する。
</StepFlowStep>
<StepFlowStep number={2} title="同意取得手続きの適切性">
インフォームドコンセントの取得手続きが適切であるか、IC簡略化の要件を満たすかを審査する。
</StepFlowStep>
<StepFlowStep number={3} title="プライバシー保護措置">
匿名化・仮名化の方法、データの安全管理措置が適切であるかを審査する。
</StepFlowStep>
<StepFlowStep number={4} title="リスクと利益のバランス">
データの二次利用によるリスク（プライバシー侵害等）と利益（AI開発による医療の質向上）のバランスが適切であるかを審査する。
</StepFlowStep>
<StepFlowStep number={5} title="社会的・倫理的妥当性">
AIの利用目的が社会的に許容されるものであるか、特定の集団に不当な不利益をもたらさないかを審査する。
</StepFlowStep>
</StepFlow>

## 同意撤回権とデータ削除

### 同意撤回の法的根拠

患者は、いつでもデータの二次利用に関する同意を撤回する権利を有する。個人情報保護法第33条は、利用停止・消去の請求権を規定している。

### AI開発における同意撤回の課題

**学習済みモデルからのデータ削除**：AIモデルが既に学習を完了した後に、特定の患者のデータの影響を除去すること（machine unlearning）は、技術的に極めて困難である。完全な削除には、そのデータを除外した上での再学習が必要になる場合がある。

**同意撤回の実務的対応**：
- 撤回時点以降のデータの利用を停止する
- 未使用の生データを削除する
- 学習済みモデルへの影響について、技術的に可能な範囲で対応する
- 対応の限界について、同意取得時にあらかじめ説明する

<Warning>
同意撤回後の対応の限界を、同意取得の時点で患者に説明しておくことが重要である。「一度AIの学習に使用されたデータの影響を完全に除去することは技術的に困難な場合がある」ことを事前に説明し、それを含めた上で同意を得る必要がある。
</Warning>

## 国際的な動向

### GDPRのデータ主体の権利

GDPRは、データ主体に以下の権利を保障している。

- **アクセス権**（第15条）：自分のデータの処理状況を知る権利
- **訂正権**（第16条）：不正確なデータの訂正を求める権利
- **消去権（忘れられる権利）**（第17条）：データの消去を求める権利
- **処理制限権**（第18条）：データの処理を制限する権利
- **データポータビリティ権**（第20条）：データを他の管理者に移転する権利
- **異議申立権**（第21条）：データの処理に異議を述べる権利

### 米国の動向

米国ではHIPAA Privacy Ruleが医療データの保護を規定しているが、適切にDe-identificationされたデータは規制対象外となる。研究目的の利用については、IRB承認またはPrivacy Boardの承認があれば、個別の同意なしに利用できる場合がある。

## 医療機関のための実務ガイド

<PromptTemplate title="データ二次利用の同意・透明性チェックリスト">
**同意取得**
- [ ] データの二次利用目的を具体的に特定した
- [ ] 適切な同意モデル（個別IC/包括同意/ダイナミックコンセント）を選択した
- [ ] 同意書に利用目的・範囲・期間・撤回方法を明記した
- [ ] 同意撤回後の対応（特にAI学習済みデータ）の限界を説明した
- [ ] IC簡略化の場合、倫理指針の要件を確認した

**倫理審査**
- [ ] 倫理審査委員会（IRB）への申請を行った
- [ ] 研究計画書に二次利用の目的・方法・リスクを記載した
- [ ] IRBの承認を取得した
- [ ] 年次報告・変更報告のスケジュールを設定した

**透明性の確保**
- [ ] データの二次利用に関する情報をウェブサイトで公開した
- [ ] 院内掲示を行った
- [ ] オプトアウトの手続きを明確にした
- [ ] 問い合わせ窓口を設置した

**データガバナンス**
- [ ] データの利用状況を記録・管理している
- [ ] 同意撤回時のデータ削除手順を整備した
- [ ] 定期的なデータ利用状況の監査を実施している
- [ ] データの保持期間を管理し、期間経過後の削除を実施している
</PromptTemplate>

<KeyTakeaway>
- 臨床データのAI開発への二次利用は、個人情報保護法上の利用目的の制限に該当し、原則として本人同意が必要
- 人を対象とする生命科学・医学系研究に関する倫理指針に基づく倫理審査（IRB）が原則として必要
- 従来型IC、包括同意、ダイナミックコンセントなど複数の同意モデルがあり、目的に応じた選択が重要
- AI学習済みモデルからの特定データ削除（machine unlearning）は技術的に困難であり、同意取得時にその限界を説明すべき
- 透明性の確保（情報公開、オプトアウト手続き、問い合わせ窓口）は患者の信頼の基盤
- GDPRは「忘れられる権利」を含む広範なデータ主体の権利を保障しており、国際展開時に考慮が必要
</KeyTakeaway>
