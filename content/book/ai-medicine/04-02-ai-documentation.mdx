## カルテ記載における3つの課題

まず、現状の課題を整理しよう。

### 課題1: 構造化の負担

診察中に頭に浮かんでいる情報は非構造的である:

- 「咳が3日前から、でも熱は昨日から」
- 「お母さんが心配そうだった」
- 「喘鳴はないけど呼吸音がちょっと粗い」
- 「とりあえず様子見でいいと思う」

これをSOAP形式に構造化するには、**認知的負荷**がかかる。

### 課題2: 記載の重複

同じ情報を複数の場所に記載する無駄:

- カルテ本文
- サマリー
- 紹介状
- 処方箋の備考

**同じことを4回書く**——これが医師の時間を奪う。

### 課題3: 法的・保険請求の要件

カルテは医療記録であると同時に、法的文書であり保険請求の根拠である。

- 「必要な記載」が漏れると査定される
- しかし「何を書けば十分か」の基準が曖昧
- 結果として過剰に詳しく書く → 時間がかかる

**AIは、この3つの課題すべてに対応できる。**

## 診察メモ → SOAP変換の基本プロンプト

カルテ記載の効率化には、2つのアプローチがある:

1. **音声入力 + AI文字起こし + SOAP構造化**（理想形、機材・環境が必要）
2. **簡潔な診察メモ → AI によるSOAP変換**（今すぐ実践可能）

今週は、**アプローチ2**——誰でも今日から使える方法——に焦点を当てる。

### 基本プロンプトの型

```
【Role】あなたは小児科医です。

【Task】以下の診察メモをSOAP形式のカルテ記載に変換してください。

【Context】
■ 診察メモ:
[ここに簡潔なメモを記載]

■ カルテ記載の目的:
- 診療録としての正確性
- 保険請求の根拠
- 次回診察時の参照

【Format】
以下のSOAP形式で記載:

**S（Subjective - 主観的情報）:**
患者・保護者の訴えを記載。「　」で引用。

**O（Objective - 客観的情報）:**
バイタルサイン、身体所見、検査結果を記載。

**A（Assessment - 評価）:**
診断名（確定診断/疑い診断）と根拠を記載。

**P（Plan - 計画）:**
処方、処置、指導、次回予約を記載。

【Constraint】
- 簡潔に（各セクション3-5行）
- 保険病名に準拠
- 処方は一般名で記載
- 「様子を見る」だけでなく、具体的な観察ポイントを明記
```

## 【実例1】3歳女児、発熱・咳嗽

### Before（診察メモ）— 1分で記載

```
3歳女児、3日前から咳、昨夜から38.5℃
咽頭発赤++、喘鳴なし、SpO2 98%
溶連菌(-)、インフル(-)
上気道炎、カルボシステイン、様子見
```

### After（SOAP）— AIによる変換（10秒）

```
**S:**
3日前から咳嗽が出現し、昨夜から38.5℃の発熱を認めた。
咳は日中・夜間ともにあり、喘鳴は聴取しないとのこと。
食欲はやや低下しているが、水分摂取は可能。

**O:**
- バイタルサイン: 体温38.3℃、SpO2 98%（室内気）
- 頭頸部: 咽頭発赤(++)、扁桃腫大なし、白苔なし
- 呼吸器: 喘鳴なし、呼吸音清
- 迅速検査: 溶連菌抗原(-)、インフルエンザ抗原(-)

**A:**
急性上気道炎（ウイルス性を疑う）
溶連菌咽頭炎、インフルエンザは検査にて否定的。

**P:**
1. カルボシステインドライシロップ 50mg/kg/日 分3 3日分
2. 対症療法として解熱薬頓用を継続
3. 水分摂取の励行を指導
4. 症状が3日以上持続する場合、または悪化する場合は再診指示
```

**時間比較**:
- 従来（手書き）: 5分
- AI支援: 1分（メモ）+ 10秒（AI変換）= **1分10秒**
- **短縮: 3分50秒**

## カルテ記載の質を担保する — 3つのチェックポイント

時間短縮は重要だが、質を犠牲にしてはならない。

AIが生成したSOAPノートを、以下の3点でチェックする:

### チェック1: 保険病名の適切性

AIが提案する診断名が、保険病名として適切か確認する。

例:
- 「ウイルス性上気道炎」→ 保険病名としては曖昧
- 「急性上気道炎」→ 保険病名として使用可

### チェック2: 処方の正確性

薬剤名、用量、用法が正確か確認する（Ch.3の検証技術を使う）。

特に注意:
- 体重換算の計算ミス
- 用法（分2, 分3）の誤り
- 禁忌薬の提案

### チェック3: 法的記載要件の充足

以下が記載されているか確認:
- インフォームド・コンセントの記録
- 副作用・リスクの説明
- 次回受診指示（緊急時の対応含む）

**この3点のチェックは30秒で完了する。それでも従来より大幅に短縮できる。**
