# 剽窃・捏造の回避と研究公正

## AI時代の新しいリスク

AIツールの普及は研究の効率化をもたらす一方で、研究公正に関する新しいリスクも生んでいます。研究不正は意図的に行われるものだけでなく、AIの特性を理解せずに使うことで無意識に発生する可能性があります。

### AIがもたらす研究公正のリスク

| リスク | 内容 | 発生メカニズム |
|--------|------|---------------|
| **意図せぬ剽窃** | AI生成テキストが既存論文と類似 | LLMが学習データの文章を再構成 |
| **架空の引用** | 存在しない論文をAIが引用 | ハルシネーション（幻覚） |
| **データの捏造** | AIが生成した「もっともらしい」データ | ユーザーの検証不足 |
| **画像の改変** | AI画像生成による研究画像の作成 | 合成画像技術の悪用 |
| **オーサーシップの希薄化** | AI依存度が高すぎて著者の知的貢献が不明確 | AIへの過度な依存 |

<Callout>
研究不正は、キャリアの終焉を意味します。論文の撤回、学位の取消、研究費の返還、所属機関からの処分に至る可能性があります。AI時代だからこそ、研究公正の原則を再確認し、新しいリスクへの対策を講じましょう。
</Callout>

## 剽窃（Plagiarism）の回避

### 剽窃の種類

1. **直接的剽窃**: 他者の文章をそのままコピー（引用なし）
2. **モザイク剽窃**: 複数の論文から文章を切り貼り
3. **自己剽窃**: 自分の既発表論文から再利用（重複出版）
4. **意訳剽窃**: 他者のアイデアを自分の言葉で書き換えるが引用しない
5. **AI経由の剽窃**: AIが学習データの文章を再構成して出力

### AI生成テキストと剽窃

LLMは大量の学術論文を学習しています。そのため、AI生成テキストが既存の論文と類似する可能性があります。これは意図的な剽窃ではありませんが、結果として剽窃と判断される可能性があります。

<Warning>
AIが生成した文章を、剽窃チェックを行わずに論文に使用するのは非常に危険です。iThenticateやTurnitin等の剽窃検出ツールで必ずチェックしてください。類似度が高い部分は自分の言葉で書き直してください。
</Warning>

### 剽窃チェックのワークフロー

<StepFlow>
  <StepFlowStep number={1} title="AI生成テキストの特定">
    論文中のAI生成テキスト（下書きを含む）を明確に把握しておく。
  </StepFlowStep>
  <StepFlowStep number={2} title="大幅なリライト">
    AI生成テキストをそのまま使用せず、自分の言葉で大幅にリライトする。構造や論理は参考にしつつ、表現は自分のものにする。
  </StepFlowStep>
  <StepFlowStep number={3} title="剽窃チェックツールの実行">
    iThenticate、Turnitin、またはPaperRaterで論文全体の類似度を確認。一般的に、20%以下の類似度が目安（引用部分を除く）。
  </StepFlowStep>
  <StepFlowStep number={4} title="高類似度部分の修正">
    類似度が高い部分を特定し、表現を修正。直接引用が必要な場合は引用符と引用番号を付ける。
  </StepFlowStep>
  <StepFlowStep number={5} title="最終チェック">
    修正後に再度剽窃チェックを実行し、類似度が許容範囲内であることを確認。
  </StepFlowStep>
</StepFlow>

## ハルシネーション（架空引用）の回避

AIの最も危険な特性の一つがハルシネーション（幻覚）です。AIは存在しない論文を、あたかも実在するかのように引用することがあります。

### ハルシネーションの特徴

- **もっともらしい**: 著者名、ジャーナル名、タイトルが実在するものに近い
- **確信を持って提示**: AIは「この論文は存在しない」とは言わない
- **細部が不正確**: DOI、年、巻号、ページが存在しない
- **混合型が多い**: 実在する著者名と実在するジャーナル名を組み合わせた架空の論文

### 引用検証のワークフロー

<PromptTemplate title="参考文献の実在確認">
以下の参考文献リストの各論文について、実在を確認してください。

[参考文献リストを貼り付け]

各論文について以下を確認してください：
1. PubMedでのPMID
2. DOIの有効性（doi.orgで解決可能か）
3. 著者名、タイトル、ジャーナル名、年の正確性
4. 論文の実在確信度（確実 / おそらく実在 / 確認不可 / 架空の疑い）

重要：この確認はAIの知識に基づくものであり、完全な保証はできません。特に「確認不可」「架空の疑い」の論文については、必ずPubMedまたはGoogle Scholarで直接検索して実在を確認してください。
</PromptTemplate>

<Warning>
AIが生成した参考文献リストは、絶対にそのまま使用しないでください。1本ずつPubMed、CrossRef、Google Scholarで実在を確認してください。ハルシネーションによる架空引用は、発覚した場合、論文の撤回につながる重大な問題です。
</Warning>

## データの捏造・改ざんの回避

### AI時代特有のリスク

- AIに「このRCTの予想される結果を生成して」と依頼し、実際のデータの代わりに使用する
- AIを使ってデータの分布を「改善」する（外れ値の除去を正当化）
- AIによるシミュレーションデータを実データとして報告する

### 対策

1. **生データの保存**: 解析に使用した生データを保存し、要求に応じて提供できるようにする
2. **解析コードの保存**: すべての統計解析コードをバージョン管理する
3. **変更履歴の記録**: データの前処理（除外基準の適用等）の全ステップを文書化する
4. **プレレジストレーション**: 解析計画を事前に登録し、事後的な変更を透明にする

## 研究画像の整合性

医学研究において、ウェスタンブロット、顕微鏡画像、臨床画像等の研究画像の改ざんは重大な不正です。

### AI画像に関する原則

| 行為 | 許容/不可 | 理由 |
|------|---------|------|
| 画像のコントラスト・明るさの均一調整 | 許容 | 全画像に同一の調整を適用 |
| 特定の部分のみの修正 | 不可 | データの改ざんに該当 |
| AIによる画像の生成・合成 | 不可 | 実データではない |
| AIによるノイズ除去 | 要開示 | 画像処理方法として記載 |

## 責任ある研究のためのチェックリスト

**剽窃チェック**
- [ ] iThenticate/Turnitinで全文チェック済み
- [ ] 類似度が許容範囲内（引用部分を除いて20%以下が目安）
- [ ] AI生成テキストを自分の言葉でリライト済み

**引用の正確性**
- [ ] すべての参考文献がPubMed/CrossRefで実在確認済み
- [ ] DOIが有効であることを確認済み
- [ ] 著者名、年、タイトルが正確

**データの整合性**
- [ ] 生データが保存されている
- [ ] 解析コードが保存されている
- [ ] データの前処理が文書化されている

**画像の整合性**
- [ ] 画像の修正が最小限かつ均一
- [ ] AI生成画像を使用していない
- [ ] 画像処理の詳細がMethodsに記載

**AI使用の開示**
- [ ] すべてのAI使用が記録されている
- [ ] MethodsまたはAcknowledgementsで開示
- [ ] 投稿先のAIポリシーに準拠

**オーサーシップ**
- [ ] すべての著者がICMJEの4基準を満たす
- [ ] AIを著者として記載していない
- [ ] すべての著者が最終稿を承認
</PromptTemplate>

## 研究倫理教育リソース

### 日本

- **APRIN（研究倫理教育eラーニング）**: 日本の多くの大学・研究機関で必須
- **日本学術振興会の研究公正ポータル**: ガイドライン、事例集
- **文部科学省の研究活動の不正行為への対応ガイドライン**

### 国際

- **ORI（Office of Research Integrity）**: 米国の研究公正に関するリソース
- **COPE（Committee on Publication Ethics）**: 出版倫理のガイドライン
- **ICMJE Recommendations**: 医学雑誌の倫理基準

<KeyTakeaway>
AI時代の研究公正は、従来のリスク（剽窃、捏造、改ざん）に加え、ハルシネーション、AI経由の意図せぬ剽窃、AI使用の不開示という新しいリスクへの対策が必要です。iThenticateによる剽窃チェック、参考文献の実在確認、AI使用の透明な開示を投稿前の必須ステップとして組み込んでください。研究の誠実さは、研究者としての最も重要な資産です。
</KeyTakeaway>
