Claude Codeは、あなたのターミナルに住み、ファイルを読み書きし、テストを走らせ、Gitを操作するAI開発者だ。

---

## 「チャットAI」の先へ

ChatGPTやClaudeのブラウザ版を使ったことがある人は多いだろう。テキストボックスに質問を入力し、AIが答えを返す。コードの書き方を聞いたり、バグの原因を相談したり、関数のリファクタリングを依頼したり。

しかし、実際の開発作業はそれだけでは完結しない。

AIが生成したコードをコピーして、エディタに貼り付けて、ファイルを保存して、テストを実行して、エラーが出たらまたチャットに戻って――。この「コピー&ペーストの往復」に、あなたはどれだけの時間を費やしているだろうか。

Claude Codeは、この往復を根本からなくすために生まれた。

---

## Claude Codeの正体

Claude Codeは、Anthropicが開発した公式のCLI（Command Line Interface）ツールである。

ブラウザのチャット画面ではなく、ターミナル（macOSならTerminal.app、WindowsならWSL）で動作する。そして、あなたのコンピュータのファイルシステムに直接アクセスできる。

```bash
$ claude
╭──────────────────────────────────────────────╮
│ ✻ Welcome to Claude Code!                     │
│                                               │
│   /help for commands, /status for settings    │
╰──────────────────────────────────────────────╯

> このプロジェクトの構造を教えて
```

この一見シンプルなインターフェースの裏側で、Claude Codeは多数のツールを使い分けている。

### できること

| 操作 | ツール | 説明 |
|------|--------|------|
| ファイル読み取り | Read | コード、画像、PDF、Jupyterノートブックを読める |
| ファイル作成 | Write | 新規ファイルを作成する |
| ファイル編集 | Edit | 既存ファイルの特定箇所を差分ベースで編集する |
| コマンド実行 | Bash | テスト、ビルド、Git操作など任意のシェルコマンド |
| コード検索 | Grep | 正規表現でコードベース全体を高速検索 |
| ファイル検索 | Glob | パターンマッチでファイルを探す |
| Web取得 | WebFetch | URLからコンテンツを取得して分析 |
| Web検索 | WebSearch | 最新情報をWeb検索で取得 |
| サブエージェント | Agent | 専門的なタスクを別のエージェントに委任 |

これらのツールを組み合わせることで、Claude Codeは実際の開発作業を行える。コードを読んで理解し、修正を加え、テストを実行して結果を確認し、問題があれば再修正する――。このサイクルを、人間が逐一指示しなくても自律的に回すことができる。

### できないこと

一方で、Claude Codeにも制約がある。

- GUI操作: デスクトップアプリケーションのボタンをクリックすることはできない
- 無制限のWeb閲覧: 認証が必要なサイト（Google Docs、Jira等）への直接アクセスは制限される
- セッション間の記憶: 明示的にメモリファイルに保存しない限り、前回のセッションの内容は忘れる
- 完全な自律性: セキュリティ上の理由から、ファイル操作やコマンド実行には人間の承認が必要

<Callout type="warning" title="AIの出力は常に検証が必要">
Claude Codeが書いたコードは「もっともらしい」が「常に正しい」わけではない。テストの実行、コードレビュー、動作確認は人間の責任で行う必要がある。これはすべてのAIコーディングツールに共通する原則だ。
</Callout>

## なぜCLIなのか

「なぜブラウザではなくターミナルなのか」という疑問は自然だ。

答えは速度と直接性にある。

ブラウザ版のChatGPTやClaudeは「対話」に最適化されている。人間がテキストを入力し、AIがテキストを返す。コードに関する相談には適しているが、実際の開発作業を行うには「コピー&ペースト」という余分なステップが常に介在する。

Claude Codeは「行動」に最適化されている。ファイルを直接読み書きし、テストを直接実行し、Gitを直接操作する。あなたの開発環境と同じ空間で動作するため、余分なステップが存在しない。

これは、医療における「電話での相談」と「直接の診察」の違いに似ている。電話では情報のやり取りしかできないが、直接の診察では触診も検査もできる。Claude Codeは、開発環境を直接「診察」できるAIなのである。

## 他のAIコーディングツールとの比較

2026年現在、AIコーディングツールは群雄割拠の時代にある。

### GitHub Copilot

エディタ内でのコード補完に特化。1行〜数行の補完は高速で、リアルタイムの「タイピング支援」として優れている。しかし、プロジェクト全体を俯瞰した大規模な変更には向かない。

### Cursor

AIネイティブなコードエディタ。エディタとAIが深く統合されており、GUIベースの操作が好みの人には使いやすい。Claude Codeとは補完的な関係にあり、実際に両方を使い分けるユーザーも多い。

### Claude Code

ターミナルベースの開発アシスタント。プロジェクト全体を理解した上で、ファイル操作・テスト実行・Git操作まで一貫して行える。大規模な変更やリファクタリング、マルチエージェント開発に強い。

| 特徴 | GitHub Copilot | Cursor | Claude Code |
|------|---------------|--------|-------------|
| インターフェース | エディタ内補完 | GUI エディタ | ターミナル CLI |
| 得意なこと | 行単位の補完 | ファイル編集 | プロジェクト全体の変更 |
| ツール実行 | 限定的 | あり | 完全対応 |
| Git操作 | なし | 限定的 | 完全対応 |
| マルチエージェント | なし | なし | 対応 |
| カスタマイズ性 | 低 | 中 | 高（Hooks/Skills/Agents/MCP） |

重要なのは、これらは「競合」ではなく「補完」の関係にあるということだ。 GitHub Copilotでリアルタイム補完を受けながら、Claude Codeで大規模なリファクタリングを行う、というワークフローは十分にあり得る。

## 医療従事者がClaude Codeを学ぶ意味

「プログラマー向けのツールでしょう？」と思うかもしれない。

しかし、以下のような場面でClaude Codeは力を発揮する。

### 臨床データ分析

電子カルテから抽出したCSVデータをPythonで分析するスクリプト。Claude Codeなら、データの構造を読み取り、適切な統計手法を選び、可視化コードまで生成してくれる。

### 患者説明用Webアプリ

疾患や治療法を患者にわかりやすく説明するためのWebアプリケーション。Claude Codeは、React/Next.jsでのフロントエンド構築からデプロイまでを一貫してサポートする。

### 研究ツールの自動化

文献検索の自動化、データ前処理のパイプライン構築、統計解析スクリプトの作成。Claude Codeは、研究ワークフローの自動化を加速する。

### 医療AIプロダクトの開発

Hoshizuのような医療AIナレッジポータルの構築。Claude Code自体が、このサイトの開発に使われている。

コードを書く医師にとって、Claude Codeは聴診器のような存在になり得る。使いこなせれば、診療（開発）の質とスピードが劇的に変わる。
