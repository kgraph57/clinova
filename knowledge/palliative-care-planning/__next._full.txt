# 医療AIワークフローガイド：緩和ケア計画立案支援

## イントロダクション

緩和ケアは、生命を脅かす疾患に関連する問題に直面している患者とその家族のQOL（生活の質）を改善することを目的としています。身体的な苦痛だけでなく、精神的、社会的、スピリチュアルな問題にも焦点を当てた全人的ケアであり、その計画立案は多岐にわたる情報を統合し、患者の価値観を尊重する複雑なプロセスです。

AIは、この複雑なプロセスにおいて、データ統合、リスク予測、コミュニケーション支援の3つの主要な側面で貢献します。電子カルテや各種評価スケールから得られる膨大なデータを迅速に解析し、潜在的な危機やニーズを早期に特定することで、多職種チームによるタイムリーで個別化されたケアの提供を支援します。本ガイドでは、AIを活用して、より質の高い、患者中心の緩和ケア計画を立案するための具体的なステップを紹介します。

---

## 基本編：初期評価と症状マネジメントの土台作り（2ステップ）

基本編では、AIを情報整理と標準的な初期介入戦略の提案に活用し、緩和ケア計画の土台を築きます。

### ステップ 1: 患者情報とニーズの初期評価

目的: 既存の臨床データから、緩和ケアにおける主要な身体的・精神的・社会的なニーズを迅速に特定し、優先順位を明確にする。

| 要素 | 詳細 |
| :--- | :--- |
| 具体的なステップ | 1. 電子カルテ、看護記録、心理評価の結果など、患者の包括的な情報をAIに入力する。 2. AIに対し、情報に基づいた主要な懸念事項（身体的、精神的、社会的）の抽出と優先順位付けを依頼する。 |
| プロンプト例 | 「以下の患者情報に基づき、緩和ケアにおける主要な身体的・精神的・社会的なニーズを特定し、優先度の高い3つの懸念事項をリストアップしてください。また、それぞれの懸念事項の根拠となる情報を明記してください。<br/><br/>[患者情報テキスト：診断名、病歴、現在の症状スコア（例：VAS、KPS）、家族構成、職業、心理社会的情報]」 |
| AIの出力例 | 優先度の高い懸念事項リスト<br/><br/>1. 疼痛コントロール（身体的）: 根拠：VASスコアが持続的に7/10。オピオイドの定時投与ではブレイクスルーペインが頻発。<br/>2. 不安と抑うつ（精神的）: 根拠：HADSスコアの不安・抑うつサブスケールが高値。夜間の不眠と「将来への恐怖」に関する発言。<br/>3. 家族の介護負担（社会的）: 根拠：主要介護者である配偶者が週7日介護に従事。介護サービス利用の拒否傾向あり。 |
| まとめ | AIは、複雑な情報を迅速に統合し、多職種チームが初期介入の焦点を定めるための客観的な基盤を提供します。これにより、見落としがちな心理社会的ニーズの早期発見が可能になります。 |

### ステップ 2: 症状マネジメントの初期戦略提案

目的: 特定された主要症状に対し、現在の治療状況を考慮したエビデンスに基づいた初期の緩和ケア戦略（薬物・非薬物）を提案させる。

| 要素 | 詳細 |
| :--- | :--- |
| 具体的なステップ | 1. ステップ1で特定された主要な懸念事項と、現在の投薬・治療情報をAIに入力する。 2. AIに対し、最新のガイドラインに基づいた症状緩和のための初期戦略を提案させる。 |
| プロンプト例 | 「特定された主要な懸念事項『疼痛コントロール』に対し、現在の投薬状況（モルヒネ徐放錠 30mg/日、レスキューにオキシコドン 5mg）を考慮した上で、推奨される薬物療法と非薬物療法の初期戦略を提案してください。特に、ブレイクスルーペインへの対応に焦点を当ててください。」 |
| AIの出力例 | 疼痛コントロール初期戦略提案<br/><br/>1. 薬物療法: モルヒネ徐放錠の定時投与量を1.5倍に増量（45mg/日）を検討。レスキュー薬の投与量は定時量の1/6（7.5mg）に増量し、投与間隔を短縮（例：1時間ごとから30分ごと）。<br/>2. 非薬物療法: 疼痛部位への温罨法または冷罨法の試行。リラクセーション技法（深呼吸、誘導イメージ）の導入と指導。ペインクリニックへのコンサルトを検討。<br/>3. モニタリング: 疼痛日誌の導入と、副作用（特に便秘、傾眠）の厳重なモニタリング。 |
| まとめ | AIは、標準的な治療プロトコルに基づき、個別化された初期介入案を迅速に生成します。これにより、治療開始までの時間を短縮し、症状の早期緩和に貢献します。 |

## 実践編：個別化と多職種連携の強化（3ステップ）

実践編では、AIをより高度な意思決定支援と多職種連携のためのツールとして活用します。

### ステップ 3: 予後予測と治療目標のすり合わせ支援

目的: 患者の臨床データに基づき、予後に関する情報を提示し、患者・家族との治療目標に関する対話（GoC: Goals of Care）を支援するための論点を整理する。

| 要素 | 詳細 |
| :--- | :--- |
| 具体的なステップ | 1. 患者の診断、病期、併存疾患、バイタルサイン、検査値などの予後に関連するデータをAIに入力する。 2. AIに対し、予後に関する情報（例：生存期間の中央値、関連する臨床指標）と、それに基づいたGoC対話の「開始の言葉」と「論点」を提案させる。 |
| プロンプト例 | 「この患者の臨床データ（[臨床データ]）に基づき、予後に関する客観的な情報（生存期間の中央値など）を提示してください。また、この情報を患者と家族に伝える際の、共感を伴う『開始の言葉』と、治療目標（延命、QOL、場所など）をすり合わせるための主要な論点を3つ提案してください。」 |
| AIの出力例 | GoC対話支援論点<br/><br/>1. 開始の言葉: 「私たちは、あなたの病状が今、非常に重要な段階にあることを理解しています。今後の治療について、あなたとご家族が最も大切にしたいことは何か、時間をかけてお話しさせてください。」<br/>2. 主要論点: (1) 優先順位の明確化: 今後数ヶ月で最も避けたいこと、最も達成したいことは何か。 (2) 治療の場所: 病院、自宅、ホスピスなど、どこで過ごしたいか。 (3) 積極的治療の範囲: どこまで延命治療を望むか、または症状緩和に焦点を移すか。 |
| まとめ | AIは、予後というデリケートな情報を扱う際のコミュニケーションを支援し、医療者が患者の価値観に基づいた意思決定を導くための構造化されたアプローチを提供します。 |

### ステップ 4: 多職種チームへの情報共有とタスク割り当て

目的: 緩和ケアチーム（医師、看護師、ソーシャルワーカー、薬剤師など）に対し、患者の最新の状況と計画に基づいた具体的なタスクを明確に割り当てる。

| 要素 | 詳細 |
| :--- | :--- |
| 具体的なステップ | 1. ステップ1～3で作成された最新のケア計画、懸念事項、GoCの要約をAIに入力する。 2. AIに対し、各職種（医師、看護師、SW、薬剤師）が直ちに行うべき具体的なアクションと期限を提案させる。 |
| プロンプト例 | 「最新の緩和ケア計画（[計画要約]）に基づき、以下の職種への具体的なタスク割り当てリストを作成してください。タスクはSMART原則（具体的、測定可能、達成可能、関連性、期限）を満たすように記述してください。」 |

# 放射線画像診断AIの現在地と未来

## 画像診断AIは「黎明期」を超えた

2020年代前半、「AIが放射線科医を置き換える」という議論が盛んでした。2026年の現実は、それとは異なる形で進化しています。AIは放射線科医を置き換えるのではなく、「AIを使う放射線科医が、使わない放射線科医を置き換える」方向に収束しつつあります。

FDA承認済みのAI医療機器は2026年3月時点で1,000件を超え、そのうち約75%が放射線科領域です。これは放射線画像という標準化されたデータ形式がAI開発に適していることを反映しています。

## 臨床導入が進む3つの領域

### 1. 緊急トリアージ（最も成熟）

脳出血、肺塞栓、大動脈解離などの緊急所見を自動検出し、読影キューの優先順位を変更するシステムが最も実用化されています。

**臨床的インパクト:**
- 緊急所見の報告時間が平均60%短縮
- 夜間・当直帯での見落とし率低下
- ワークフロー効率の改善

代表的な製品として、Aidocの頭部CT出血検出（感度95%以上）やViz.aiの大血管閉塞検出（脳卒中チーム起動を20分以上短縮）があります。

### 2. 定量解析（急成長中）

従来は「主観的」だった画像評価を定量化するAIが増えています。

- **骨密度測定**: 日常的なCTから骨密度を推算し、骨粗鬆症スクリーニング
- **冠動脈石灰化スコア**: 非造影CTからCACスコアを自動算出
- **脳容積測定**: 認知症の経時的変化を定量追跡
- **肺結節測定**: RECIST基準に基づく腫瘍径の自動測定と経時比較

### 3. スクリーニング補助（発展途上）

マンモグラフィのAI読影は欧州で大規模臨床試験が進行中です。スウェーデンのMASSIVE試験（8万人規模）では、AI支援読影が二重読影と同等以上の精度を示し、読影医の負担を半減させる可能性が示されました。

## 導入時の落とし穴

### バイアスの問題

学習データの偏りは深刻です。米国の大規模データセットで学習したモデルが、アジア人患者で精度低下する事例が報告されています。自施設のデータでバリデーションを行うことが不可欠です。

### アラート疲れ

偽陽性が多すぎると、臨床医がアラートを無視するようになります。感度と特異度のバランス、アラートの提示方法の設計が重要です。

### 統合コスト

既存のPACS/RISとの統合には技術的・制度的な課題があります。DICOM標準への準拠は前提ですが、実際の導入には数か月のカスタマイズが必要なケースも多いです。

## AIと放射線科医の協働モデル

最も生産性が高いモデルは「AIファースト・リード」です。

1. **AI前処理**: 画像のセグメンテーション、定量測定、異常候補の抽出
2. **医師確認**: AIの所見を参照しながら最終読影
3. **フィードバック**: 医師の修正がAIの学習データに還元

このモデルでは、放射線科医の役割は「画像を見る人」から「AIの出力を検証し、臨床的文脈を加える人」に変化します。

## 今後の展望

- **マルチモーダルAI**: 画像＋電子カルテ＋検査データを統合した診断支援
- **術前シミュレーション**: 3D再構成とAI解析の組み合わせによる手術計画
- **リアルタイム超音波ガイド**: 超音波プローブを握る非専門医をAIがリアルタイム支援

画像診断AIは「使うかどうか」ではなく「どう使うか」のフェーズに入っています。臨床医としては、AIの限界を理解したうえで、自施設のワークフローに最適な形で導入することが重要です。

## 使い方

2. 炎症がある場合、Albは栄養指標として信頼性が低下する（CRPと合わせて評価）
3. 通常時体重からの変化率が重要な指標
