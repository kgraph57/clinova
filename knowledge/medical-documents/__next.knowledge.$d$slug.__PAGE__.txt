# 診断書・紹介状作成支援：AIで医療文書作成を効率化・標準化

## はじめに

診断書、紹介状、退院サマリーといった医療文書の作成は、医師の日常業務において大きな割合を占める、重要でありながら時間のかかる作業です。
これらの文書は、患者の法的な権利を守り、施設間の連携をスムーズにし、継続的なケアの質を担保する上で不可欠な役割を担っています。

しかし、多忙な臨床業務の中で、これらの文書を正確に、分かりやすく作成することは、多くの医師にとって負担となっています。
「何を書けばいいのか迷う」「定型的な文章の入力に時間がかかる」「記載内容の抜け漏れが心配」といった悩みは、誰もが経験するものです。

このガイドでは、AIを活用して、診断書や紹介状などの医療文書作成を劇的に効率化し、質を向上させる方法を紹介します。
AIを優秀なメディカルライターとして活用し、文書作成の負担を軽減することで、より多くの時間を患者と向き合うために使いましょう。

![医療文書作成ワークフロー](/images/guides/medical-documents/medical_documents_workflow.png)

---

## なぜAI支援が必要なのか？

### 医療文書作成の課題

診療時間外に多くの時間を費やしているという時間的負担があります。
医師によって記載内容の詳しさや形式が異なり、受け取り手が混乱することがあります。
記載すべき重要な情報、例えばアレルギー歴や主要な現病歴などが欠落する可能性があります。
簡潔かつ的確に、必要な情報を過不足なく伝える文章を作成するのは難しいものです。
特に、休職診断書や法的文書など、患者の人生に大きな影響を与える文書作成は精神的な負担が大きいです。

### AI支援の効果

AIを活用することで、これらの課題を克服し、文書作成の質と効率を向上させることができます。

患者情報や目的を指示するだけで、AIが文書のドラフト（下書き）を数秒で作成します。
目的に応じて、紹介、返信、公的機関への提出など、適切なフォーマットで構造化された文書を作成します。
記載すべき項目を網羅したテンプレートをAIが提示することで、重要な情報の欠落を防ぎます。
専門的で、誰が読んでも分かりやすい、洗練された文章を生成します。
患者の背景、例えば年齢、職業、社会的背景などを考慮した、個別性の高い文書作成を支援します。

AIが生成した文書は、あくまで「下書き（ドラフト）」です。
最終的な内容の正確性、妥当性、法的な責任は、すべて署名する医師にあります。

AIの生成結果を、絶対にそのままコピー＆ペーストして使用しないでください。
必ず、以下の点をご自身の目で確認・修正してください。

患者の氏名、生年月日、病名、日付、検査データなどの客観的な情報が正確か、事実確認を行ってください。
診断の根拠、治療内容、今後の見通しなどが、医学的に見て妥当か、医学的妥当性を検証してください。
その患者固有の状況や背景が、適切に反映されているか、個別性を確認してください。
AIが生成した文章のニュアンスが、伝えたい意図と合っているか、ニュアンスを確認してください。

AIは強力なツールですが、万能ではありません。
最終的な責任は、プロフェッショナルであるあなた自身にあることを、常に心に留めておいてください。

# ステップ1: 診断書のドラフトを生成する

## 診断書の基本構成

診断書は、患者の健康状態について、医師が専門的な見地から証明する公的な文書です。
その目的は、休職、保険請求、公的支援の申請など多岐にわたります。
どのような目的であれ、診断書には共通して記載すべき基本要素があります。

まず、患者情報として、氏名、生年月日、性別、住所を記載します。
次に、診断名として、主病名と関連する副病名を記載します。
ICD-10コードを併記することもあります。
発症・診断日として、症状が始まった日と診断が確定した日を記載します。
診断の根拠として、症状、身体所見、検査結果など、診断に至った客観的な根拠を記載します。
治療内容と経過として、これまで行ってきた治療、例えば投薬や手術などとその効果を記載します。
現在の状況と今後の見通しとして、現在の症状の程度、治療状況、今後の回復の見込みなどを記載します。
特記事項として、目的に応じて、就労に関する意見、必要な配慮、後遺症の有無などを記載します。
最後に、作成日、病院名、医師名、押印を記載します。

これらの要素を網羅し、目的に応じて記載の詳しさを調整することが重要です。

---

## 診断書作成の基本プロンプト

このプロンプトは、患者の基本情報と診断書の目的をAIに伝えることで、上記の構成要素を網羅した診断書のドラフトを生成させるためのものです。

### 📋 診断書基本プロンプト（コピペ用）

```
あなたは、様々な診療科の経験が豊富な医師です。以下の患者情報に基づき、指定された目的の診断書のドラフト（下書き）を作成してください。

【患者情報】
* 氏名: [氏名]
* 生年月日: [生年月日]
* 主病名: [主病名]
* 診断日: [診断日]
* 診断根拠: [症状、所見、検査結果などを簡潔に]
* 治療経過: [これまでの治療内容を簡潔に]
* 現在の状況: [現在の症状や状態を簡潔に]

【診断書の目的】
[例: 会社に提出し、病気療養のための休職を申請する]

【出力形式】
一般的な診断書のフォーマットに従って、丁寧な言葉遣いで記述してください。特に、診断書の目的に関連する「今後の見通し」や「必要な配慮」については、具体的に記述してください。
```

# ステップ2: 紹介状・返信状を作成する

## 医療連携の要、SOAP形式

紹介状（診療情報提供書）やその返信状は、患者の情報を医療機関の間で正確に共有し、継続的で質の高い医療を提供するための、非常に重要なコミュニケーションツールです。

これらの文書を作成する上で、情報を構造化し、抜け漏れなく伝えるための世界標準のフレームワークがSOAP（ソープ）です。

SはSubjective（主観的情報）を表します。
患者の訴え（主訴）、現病歴、既往歴、アレルギー歴などが含まれます。
OはObjective（客観的情報）を表します。
身体所見、バイタルサイン、検査結果、画像所見などが含まれます。
AはAssessment（評価）を表します。
SとOから導き出される、医師による評価、考察、アセスメント、診断が含まれます。
PはPlan（計画）を表します。
Aに基づいて立案される、今後の治療計画、検査計画、紹介の目的などが含まれます。

このSOAP形式を用いることで、受け取った医師は患者の状態を迅速かつ正確に把握することができます。

---

## 紹介状・返信状作成の基本プロンプト

このプロンプトは、SOAPの各項目に関する情報をAIに与え、構造化された質の高い紹介状や返信状のドラフトを生成させるためのものです。

### 📋 紹介状基本プロンプト（コピペ用）

```
あなたは、経験豊富な総合内科医です。以下の患者情報（SOAP形式）に基づき、指定された宛先の専門医への紹介状のドラフトを作成してください。

【宛先】
[例: 〇〇大学病院 呼吸器内科 教授 〇〇先生 御侍史]

【紹介目的】
[例: 精査・加療のため]

【患者情報】
* 患者: [氏名、年齢、性別]
* 紹介元: [あなたの病院・クリニック名、あなたの名前]

【SOAP】
* S (Subjective):
  * 主訴: 
  * 現病歴: 
  * 既往歴: 
  * 生活歴・家族歴: 
  * 内服薬: 

* O (Objective):
  * 身体所見: 
  * 検査データ: 
  * 画像所見: 

* A (Assessment):
  * [SとOを統合したアセスメント、考えられる鑑別診断など]

* P (Plan):
  * [紹介に至った経緯、依頼したい内容など]

【出力形式】
一般的な紹介状のフォーマットに従い、専門医に対する敬意を払った丁寧な言葉遣いで記述してください。特に、AssessmentとPlanを明確に記述してください。
```

## 実践例: 肺癌疑いの患者紹介

### 入力プロンプト

```
あなたは、経験豊富な総合内科医です。以下の患者情報（SOAP形式）に基づき、指定された宛先の専門医への紹介状のドラフトを作成してください。

【宛先】
メディカル大学病院 呼吸器内科 教授 佐藤 一郎 先生 御侍史

【紹介目的】
肺癌の疑い。確定診断および治療方針決定のため。

【患者情報】
* 患者: 鈴木 次郎 様 (72歳・男性)
* 紹介元: まなぶ内科クリニック 院長 まなぶ 太郎

【SOAP】
* S: 2ヶ月前からの乾性咳嗽と、最近出現した血痰を主訴に来院。40年来の喫煙歴（20本/日）。
```

# ステップ1: 退院サマリーを作成する

## 継続的なケアのための情報集約

退院サマリー（退院時要約）は、一人の患者の入院中の情報を集約し、退院後のケアを担当するかかりつけ医、訪問看護師、介護スタッフなど、関係者全員が患者の状態を正確に把握するための、極めて重要な文書です。

質の高い退院サマリーは、退院後の再入院を防ぎ、シームレスな地域連携を実現するための鍵となります。
しかし、入院期間が長くなればなるほど、その情報を簡潔かつ網羅的にまとめるのは大変な作業です。
AIは、この複雑な情報整理と文書化のプロセスを大幅に効率化します。

---

# ステップ2: 目的別の診断書を作成する

## 文書の「受け取り手」を意識する

診断書は、その目的によって記載すべき内容の重点や、求められる情報の種類が異なります。
文書の「受け取り手」が誰で、その人が何を知りたいのかを意識することが、質の高い診断書を作成する上での鍵となります。

AIに指示を出す際も、この「目的」と「受け取り手」を明確に伝えることで、より適切で実用的なドラフトを生成させることができます。

---

## 1. 身体疾患による休職診断書

目的: 身体的な病気の治療に専念するため、一定期間の休職が必要であることを会社に証明する。
受け取り手: 会社の人事・労務担当者、産業医
ポイント: なぜ就労が困難なのか、その医学的根拠と、必要な休養期間の見込みを明確に記述する。

# ステップ1: 介護保険主治医意見書を作成する

## 多角的な視点が求められる意見書

介護保険主治医意見書は、市区町村が要介護認定を行う際に、申請者の心身の状態について主治医が医学的な見地から意見を述べるための、非常に重要な文書です。
単なる診断書とは異なり、病気や怪我そのものだけでなく、それが日常生活動作（ADL）や認知機能にどのような影響を与えているかという、より生活に密着した視点が求められます。

記載項目が多岐にわたり、作成に手間がかかるため、多くの医師が負担に感じています。
AIを活用することで、患者の情報を基に、各項目に対応する記述のドラフトを効率的に作成することができます。

---

## 主治医意見書作成支援プロンプト

# ステップ2: 配慮の行き届いた文章表現をAIに相談する

## 言葉の選び方が、患者の未来を左右する

医療文書、特に診断書に書かれる言葉は、単なる事実の記述以上の重みを持ちます。
その一言一句が、患者さんやそのご家族の感情、時にはその後の人生の選択にまで、大きな影響を与えることがあります。

例えば、「治癒の見込みはない」と書くか、「症状のコントロールを目標とします」と書くか。
事実は同じでも、受け手の印象は大きく異なります。

私たちは、医学的な正確性を保ちつつも、患者さんの希望を奪わず、前向きな姿勢を支えるような、温かみのある言葉を選ぶ必要があります。
AIは、このような繊細なニュアンスの表現を考える上での、良き相談相手となります。

---

## 表現を洗練させるためのプロンプト

このプロンプトは、あなたが書いた文章のドラフトをAIに提示し、より患者の心情に配慮した、あるいは、より正確で誤解の少ない表現への言い換えを提案させるためのものです。

### 📋 表現相談プロンプト（コピペ用）

```
あなたは、コミュニケーションの専門家であり、言葉の選び方が患者の心理に与える影響を深く理解しているカウンセラーです。以下の診断書の記述について、より患者さんの心情に配慮した、前向きで温かみのある表現への言い換えを3パターン提案してください。

【元の文章】
[ここに、あなたが書いた、あるいはAIが生成した文章のドラフトを貼り付ける]

【背景・文脈】
* 患者: [患者の年齢、性別、性格など]
* 病名: [病名]
* 状況: [なぜこの記述が必要なのか、その背景]

【求めるニュアンス】
* [例: 希望を失わせず、治療に前向きになれるような表現]
* [例: できないことではなく、できることに焦点を当てた表現]
* [例: 突き放すのではなく、寄り添う姿勢が伝わる表現]
```

## 実践例: 進行がん患者の診断書

### 入力プロンプト

```
あなたは、コミュニケーションの専門家であり、言葉の選び方が患者の心理に与える影響を深く理解している緩和ケア医です。以下の診断書の記述について、より患者さんの心情に配慮した、前向きで温かみのある表現への言い換えを3パターン提案してください。

【元の文章】
「全身に転移を認めるため、根治は困難である。今後は、化学療法による延命と、症状緩和が治療の中心となる。」

【背景・文脈】
* 患者: 65歳男性、前向きな性格だが、最近少し元気をなくしている。
* 病名: 進行性肺がん（ステージIV）
* 状況: 本人への告知後の、生命保険の申請に必要な診断書の記述。

【求めるニュアンス】
* 「根治は困難」という厳しい現実を伝えつつも、絶望感を与えない表現。
* 今後の治療が、単なる「延命」ではなく、QOL（生活の質）を維持するための積極的なものであることが伝わる表現。
```

### AIの出力例

```
承知いたしました。厳しい状況の中でも、患者様が希望を失わず、これからの治療に前向きに取り組めるような表現を提案します。

### 言い換え提案

パターン1（目標の再設定を強調する表現）
> がんが体の広い範囲に見られるため、病気を完全に消し去ることを目指すのではなく、がんと共存しながら、患者様ご自身の『生活の質』を最大限に高めることを、新たな治療目標とします。今後は、化学療法によってがんの進行を穏やかにし、痛みなどの症状を和らげることで、これまでと変わらない穏やかな日常をできるだけ長く維持できるよう、チーム一丸となってサポートして参ります。

パターン2（患者の「時間」を大切にする視点）
> 病気の性質上、完全に治すことは難しい状況ですが、残された貴重な時間を、より豊かで意味のあるものにするための治療が可能です。今後は、化学療法によって病気の進行をコントロールし、症状緩和を積極的に行うことで、ご家族と過ごす時間や、ご自身の趣味を楽しむ時間など、〇〇様にとって大切な時間を守ることを最優先の目標といたします。

パターン3（よりシンプルで、力強い表現）
> 病気は進行しておりますが、治療の選択肢がなくなったわけではありません。これからの治療の主役は、病気を治すことではなく、〇〇様ご自身です。化学療法や緩和ケアを組み合わせることで、体の辛さを和らげ、自分らしい生活を続けることは十分に可能です。私たちは、そのための最善の医療を提供することをお約束します。

【解説】
これらの表現は、いずれも「できないこと（根治）」ではなく「できること（症状緩和、QOL維持）」に焦点を当てています。
「延命」という言葉を避け、「穏やかな日常」「豊かな時間」「自分らしい生活」といった、患者様が具体的にイメージできるポジティブな言葉に置き換えることで、今後の治療に対する前向きな動機付けを促すことを意図しています。
```

# 診断書・紹介状作成支援：AIで医療文書作成を効率化・標準化

## はじめに

診断書、紹介状、退院サマリーといった医療文書の作成は、医師の日常業務において大きな割合を占める、重要でありながら時間のかかる作業です。
これらの文書は、患者の法的な権利を守り、施設間の連携をスムーズにし、継続的なケアの質を担保する上で不可欠な役割を担っています。

しかし、多忙な臨床業務の中で、これらの文書を正確に、分かりやすく作成することは、多くの医師にとって負担となっています。
「何を書けばいいのか迷う」「定型的な文章の入力に時間がかかる」「記載内容の抜け漏れが心配」といった悩みは、誰もが経験するものです。

このガイドでは、AIを活用して、診断書や紹介状などの医療文書作成を劇的に効率化し、質を向上させる方法を紹介します。
AIを優秀なメディカルライターとして活用し、文書作成の負担を軽減することで、より多くの時間を患者と向き合うために使いましょう。

![医療文書作成ワークフロー](/images/guides/medical-documents/medical_documents_workflow.png)

---

## なぜAI支援が必要なのか？

### 医療文書作成の課題

診療時間外に多くの時間を費やしているという時間的負担があります。
医師によって記載内容の詳しさや形式が異なり、受け取り手が混乱することがあります。
記載すべき重要な情報、例えばアレルギー歴や主要な現病歴などが欠落する可能性があります。
簡潔かつ的確に、必要な情報を過不足なく伝える文章を作成するのは難しいものです。
特に、休職診断書や法的文書など、患者の人生に大きな影響を与える文書作成は精神的な負担が大きいです。

### AI支援の効果

AIを活用することで、これらの課題を克服し、文書作成の質と効率を向上させることができます。

患者情報や目的を指示するだけで、AIが文書のドラフト（下書き）を数秒で作成します。
目的に応じて、紹介、返信、公的機関への提出など、適切なフォーマットで構造化された文書を作成します。
記載すべき項目を網羅したテンプレートをAIが提示することで、重要な情報の欠落を防ぎます。
専門的で、誰が読んでも分かりやすい、洗練された文章を生成します。
患者の背景、例えば年齢、職業、社会的背景などを考慮した、個別性の高い文書作成を支援します。

診断書の基本構成を理解します。
AIを使って、様々な状況に応じた診断書のドラフトを作成するプロンプトを習得します。
紹介状・返信状の基本構成（SOAP形式）を理解します。
AIを使って、質の高い紹介状・返信状を効率的に作成するプロンプトを習得します。
退院サマリーや介護保険主治医意見書など、より複雑な文書作成を支援させる方法を学びます。

## 非常に重要な注意事項

AIが生成した文書は、あくまで「下書き（ドラフト）」です。
最終的な内容の正確性、妥当性、法的な責任は、すべて署名する医師にあります。

AIの生成結果を、絶対にそのままコピー＆ペーストして使用しないでください。
必ず、以下の点をご自身の目で確認・修正してください。

患者の氏名、生年月日、病名、日付、検査データなどの客観的な情報が正確か、事実確認を行ってください。
診断の根拠、治療内容、今後の見通しなどが、医学的に見て妥当か、医学的妥当性を検証してください。
その患者固有の状況や背景が、適切に反映されているか、個別性を確認してください。
AIが生成した文章のニュアンスが、伝えたい意図と合っているか、ニュアンスを確認してください。

AIは強力なツールですが、万能ではありません。
最終的な責任は、プロフェッショナルであるあなた自身にあることを、常に心に留めておいてください。

# ステップ1: 診断書のドラフトを生成する

## 診断書の基本構成

診断書は、患者の健康状態について、医師が専門的な見地から証明する公的な文書です。
その目的は、休職、保険請求、公的支援の申請など多岐にわたります。
どのような目的であれ、診断書には共通して記載すべき基本要素があります。

まず、患者情報として、氏名、生年月日、性別、住所を記載します。
次に、診断名として、主病名と関連する副病名を記載します。
ICD-10コードを併記することもあります。
発症・診断日として、症状が始まった日と診断が確定した日を記載します。
診断の根拠として、症状、身体所見、検査結果など、診断に至った客観的な根拠を記載します。
治療内容と経過として、これまで行ってきた治療、例えば投薬や手術などとその効果を記載します。
現在の状況と今後の見通しとして、現在の症状の程度、治療状況、今後の回復の見込みなどを記載します。
特記事項として、目的に応じて、就労に関する意見、必要な配慮、後遺症の有無などを記載します。
最後に、作成日、病院名、医師名、押印を記載します。

これらの要素を網羅し、目的に応じて記載の詳しさを調整することが重要です。

---

## 診断書作成の基本プロンプト

このプロンプトは、患者の基本情報と診断書の目的をAIに伝えることで、上記の構成要素を網羅した診断書のドラフトを生成させるためのものです。

### 📋 診断書基本プロンプト（コピペ用）

```
あなたは、様々な診療科の経験が豊富な医師です。以下の患者情報に基づき、指定された目的の診断書のドラフト（下書き）を作成してください。

【患者情報】
* 氏名: [氏名]
* 生年月日: [生年月日]
* 主病名: [主病名]
* 診断日: [診断日]
* 診断根拠: [症状、所見、検査結果などを簡潔に]
* 治療経過: [これまでの治療内容を簡潔に]
* 現在の状況: [現在の症状や状態を簡潔に]

【診断書の目的】
[例: 会社に提出し、病気療養のための休職を申請する]

【出力形式】
一般的な診断書のフォーマットに従って、丁寧な言葉遣いで記述してください。特に、診断書の目的に関連する「今後の見通し」や「必要な配慮」については、具体的に記述してください。
```

# ステップ2: 紹介状・返信状を作成する

## 医療連携の要、SOAP形式

紹介状（診療情報提供書）やその返信状は、患者の情報を医療機関の間で正確に共有し、継続的で質の高い医療を提供するための、非常に重要なコミュニケーションツールです。

これらの文書を作成する上で、情報を構造化し、抜け漏れなく伝えるための世界標準のフレームワークがSOAP（ソープ）です。

SはSubjective（主観的情報）を表します。
患者の訴え（主訴）、現病歴、既往歴、アレルギー歴などが含まれます。
OはObjective（客観的情報）を表します。
身体所見、バイタルサイン、検査結果、画像所見などが含まれます。
AはAssessment（評価）を表します。
SとOから導き出される、医師による評価、考察、アセスメント、診断が含まれます。
PはPlan（計画）を表します。
Aに基づいて立案される、今後の治療計画、検査計画、紹介の目的などが含まれます。

このSOAP形式を用いることで、受け取った医師は患者の状態を迅速かつ正確に把握することができます。

```
【宛先】
[例: 〇〇大学病院 呼吸器内科 教授 〇〇先生 御侍史]

【紹介目的】
[例: 精査・加療のため]

【患者情報】
* 患者: [氏名、年齢、性別]
* 紹介元: [あなたの病院・クリニック名、あなたの名前]

【SOAP】
* S (Subjective):
  * 主訴: 
  * 現病歴: 
  * 既往歴:

  * 内服薬:

  * 身体所見:
```

# ステップ1: 退院サマリーを作成する

## 継続的なケアのための情報集約

退院サマリー（退院時要約）は、一人の患者の入院中の情報を集約し、退院後のケアを担当するかかりつけ医、訪問看護師、介護スタッフなど、関係者全員が患者の状態を正確に把握するための、極めて重要な文書です。

質の高い退院サマリーは、退院後の再入院を防ぎ、シームレスな地域連携を実現するための鍵となります。
しかし、入院期間が長くなればなるほど、その情報を簡潔かつ網羅的にまとめるのは大変な作業です。
AIは、この複雑な情報整理と文書化のプロセスを大幅に効率化します。

---

## 実践例: 心不全増悪による入院患者の退院サマリー

### 入力プロンプト

```
あなたは、経験豊富な循環器内科医です。以下の患者の入院経過に関する情報を基に、退院後のケアを担当するかかりつけ医向けの、詳細な退院サマリーのドラフトを作成してください。

【患者情報】
* 氏名: 高橋 清 様
* 年齢・性別: 82歳・女性
* 入院日: 2025年11月28日
* 退院日: 2025年12月13日

【入院情報】
* 主病名: 急性心不全（HFrEFの急性増悪）
* 副病名: 慢性心房細動、2型糖尿病、慢性腎臓病（CKD G3b）
* 入院までの経緯: 1週間前からの呼吸困難と下腿浮腫の増悪を主訴に救急外来を受診。胸部X線でうっ血所見、BNP 1200 pg/mLと高値のため、心不全増悪と診断し緊急入院。
* 入院中の経過: フロセミド静注にて利尿を図り、体重は5kg減少。呼吸困難および浮腫は著明に改善。心不全治療薬（ARNI、β遮断薬、MRA）を再調整し、SGLT2阻害薬を新規導入。糖尿病・腎機能については大きな変動なく経過。
* 退院時処方: サクビトリルバルサルタン、ビソプロロール、スピロノラクトン、ダパグリフロジン、アピキサバン、インスリングラルギン
* 退院後の注意点: 厳格な塩分制限（6g/日未満）と水分管理。毎日体重を測定し、2kg以上の増加があれば連絡するよう指導。次回の当科外来は1ヶ月後。
```

# ステップ2: 目的別の診断書を作成する

## 文書の「受け取り手」を意識する

診断書は、その目的によって記載すべき内容の重点や、求められる情報の種類が異なります。
文書の「受け取り手」が誰で、その人が何を知りたいのかを意識することが、質の高い診断書を作成する上での鍵となります。

AIに指示を出す際も、この「目的」と「受け取り手」を明確に伝えることで、より適切で実用的なドラフトを生成させることができます。

## 1. 身体疾患による休職診断書

目的: 身体的な病気の治療に専念するため、一定期間の休職が必要であることを会社に証明する。
受け取り手: 会社の人事・労務担当者、産業医
ポイント: なぜ就労が困難なのか、その医学的根拠と、必要な休養期間の見込みを明確に記述する。

ポイント: 傷病と業務との間の因果関係について、客観的な事実に基づいて医学的な意見を述べることが最も重要。

```

【出力形式】
```

# ステップ1: 介護保険主治医意見書を作成する

## 多角的な視点が求められる意見書

介護保険主治医意見書は、市区町村が要介護認定を行う際に、申請者の心身の状態について主治医が医学的な見地から意見を述べるための、非常に重要な文書です。
単なる診断書とは異なり、病気や怪我そのものだけでなく、それが日常生活動作（ADL）や認知機能にどのような影響を与えているかという、より生活に密着した視点が求められます。

記載項目が多岐にわたり、作成に手間がかかるため、多くの医師が負担に感じています。
AIを活用することで、患者の情報を基に、各項目に対応する記述のドラフトを効率的に作成することができます。

# ステップ2: 配慮の行き届いた文章表現をAIに相談する

## 言葉の選び方が、患者の未来を左右する

医療文書、特に診断書に書かれる言葉は、単なる事実の記述以上の重みを持ちます。
その一言一句が、患者さんやそのご家族の感情、時にはその後の人生の選択にまで、大きな影響を与えることがあります。

例えば、「治癒の見込みはない」と書くか、「症状のコントロールを目標とします」と書くか。
事実は同じでも、受け手の印象は大きく異なります。

私たちは、医学的な正確性を保ちつつも、患者さんの希望を奪わず、前向きな姿勢を支えるような、温かみのある言葉を選ぶ必要があります。
AIは、このような繊細なニュアンスの表現を考える上での、良き相談相手となります。

---

## 表現を洗練させるためのプロンプト

このプロンプトは、あなたが書いた文章のドラフトをAIに提示し、より患者の心情に配慮した、あるいは、より正確で誤解の少ない表現への言い換えを提案させるためのものです。

### 📋 表現相談プロンプト（コピペ用）

```
あなたは、コミュニケーションの専門家であり、言葉の選び方が患者の心理に与える影響を深く理解しているカウンセラーです。以下の診断書の記述について、より患者さんの心情に配慮した、前向きで温かみのある表現への言い換えを3パターン提案してください。

【元の文章】
[ここに、あなたが書いた、あるいはAIが生成した文章のドラフトを貼り付ける]

【背景・文脈】
* 患者: [患者の年齢、性別、性格など]
* 病名: [病名]
* 状況: [なぜこの記述が必要なのか、その背景]

【求めるニュアンス】
* [例: 希望を失わせず、治療に前向きになれるような表現]
* [例: できないことではなく、できることに焦点を当てた表現]
* [例: 突き放すのではなく、寄り添う姿勢が伝わる表現]
```

## 実践例: 進行がん患者の診断書

### 入力プロンプト

```
あなたは、コミュニケーションの専門家であり、言葉の選び方が患者の心理に与える影響を深く理解している緩和ケア医です。以下の診断書の記述について、より患者さんの心情に配慮した、前向きで温かみのある表現への言い換えを3パターン提案してください。

【元の文章】
「全身に転移を認めるため、根治は困難である。今後は、化学療法による延命と、症状緩和が治療の中心となる。」

【背景・文脈】
* 患者: 65歳男性、前向きな性格だが、最近少し元気をなくしている。
* 病名: 進行性肺がん（ステージIV）
* 状況: 本人への告知後の、生命保険の申請に必要な診断書の記述。

【求めるニュアンス】
* 「根治は困難」という厳しい現実を伝えつつも、絶望感を与えない表現。
* 今後の治療が、単なる「延命」ではなく、QOL（生活の質）を維持するための積極的なものであることが伝わる表現。
```

### AIの出力例

```
承知いたしました。厳しい状況の中でも、患者様が希望を失わず、これからの治療に前向きに取り組めるような表現を提案します。

### 言い換え提案

パターン1（目標の再設定を強調する表現）
> がんが体の広い範囲に見られるため、病気を完全に消し去ることを目指すのではなく、がんと共存しながら、患者様ご自身の『生活の質』を最大限に高めることを、新たな治療目標とします。今後は、化学療法によってがんの進行を穏やかにし、痛みなどの症状を和らげることで、これまでと変わらない穏やかな日常をできるだけ長く維持できるよう、チーム一丸となってサポートして参ります。

パターン2（患者の「時間」を大切にする視点）
```

# Ambient AI：診察室の会話が自動でカルテになる時代

## Ambient Clinical Documentationとは

Ambient Clinical Documentation（ACD）は、診察室での医師と患者の会話をAIがリアルタイムで聴き取り、構造化されたカルテ記録に自動変換するテクノロジーです。医師はパソコンに向かってタイプする代わりに、患者と目を合わせて会話するだけでカルテが完成します。

米国では2024〜2025年にかけて爆発的に普及し、2026年時点で大手病院チェーンの過半数が何らかのACD製品を導入しています。

## 主要プレイヤー

| 製品 | 開発元 | 特徴 |
|------|--------|------|
| DAX Copilot | Microsoft/Nuance | Epic統合、最大シェア |
| Abridge | Abridge | 根拠文の表示が強み |
| Nabla Copilot | Nabla | EU市場に強い |
| Suki | Suki | 音声コマンド統合 |
| DeepScribe | DeepScribe | 専門科対応が充実 |

## 何ができるのか

### 基本機能

1. **会話の文字起こし**: 医師と患者の会話をリアルタイムで文字化
2. **話者分離**: 医師の発言と患者の発言を自動で分離
3. **SOAP変換**: 会話内容からSubjective、Objective、Assessment、Planを自動生成
4. **医学用語の正規化**: 口語表現を医学用語に変換（「お腹が痛い」→「腹痛」）
5. **コード提案**: ICD-10コードや処置コードの自動提案

### 高度な機能

- **検査オーダーの提案**: 会話内容から必要な検査を推測して提案
- **処方の事前入力**: 会話で言及された薬剤の処方を下書き
- **患者向けサマリー**: 患者に渡す平易な言葉の診療要約を自動生成

## 米国での導入効果

### 定量的効果

- 医師の満足度: 導入後85%が「業務改善」と回答

### 定性的効果

- 患者との目線の合う時間が増え、信頼関係が改善
- カルテの網羅性が向上（会話全体が記録されるため漏れが減少）
- 医師の燃え尽き症候群スコアの改善

# AI文書作成で診療時間を取り戻す

## 文書業務が奪う診療時間

医師の業務時間のうち、文書作成に費やされる時間は平均30〜40%と報告されています。2024年の医師の働き方改革で労働時間の上限規制が始まりましたが、診療の質を落とさずに文書業務を効率化することは、すべての医師にとって喫緊の課題です。

AIを活用した文書作成は、単に「早く書ける」だけでなく、「漏れなく書ける」「一貫性のある文書が書ける」という品質面のメリットもあります。

## AI文書作成の3段階

### レベル1: テンプレート生成（即日導入可能）

最も手軽なのは、テンプレートの自動生成です。

**退院サマリーの例:**
- 入院日、退院日、主病名、手術名を入力
- AIが標準的な退院サマリーの骨格を生成
- 医師が内容を確認・修正

この段階では、AIは「下書き」を作るだけです。医師の確認は必須ですが、白紙から書き始めるよりも大幅に時間を短縮できます。

### レベル2: カルテ情報からの自動要約（導入進行中）

電子カルテの経過記録から、退院サマリーや紹介状を自動要約するシステムが普及し始めています。

**メリット:**
- 入院経過の記録漏れを防ぐ
- 時系列の整理が正確
- 検査結果の変遷を自動でまとめる

**注意点:**
- カルテの記載が不十分だと、要約も不十分になる
- 「書いていないこと」はAIにも分からない
- コピー＆ペーストの連鎖で誤情報が混入するリスク

### レベル3: Ambient Documentation（最先端）

診察室での医師と患者の会話をリアルタイムで文字起こしし、SOAP形式のカルテに自動変換するシステムです。Nuance DAXやAbridge、Nabla Copilotなどが代表的です。

**効果（米国での報告）:**
- カルテ記載時間が50%以上短縮
- 患者との対話時間が増加
- 医師の燃え尽き症候群スコアが改善

日本語対応はまだ発展途上ですが、2026年に入り複数のベンダーが日本市場に参入しています。

## 文書別の活用ポイント

### 紹介状

紹介状で最も重要なのは「なぜ紹介するのか」と「何をしてほしいのか」の明確化です。AIに以下を入力すると、適切な紹介状が生成できます。

- 紹介目的（精査依頼、治療依頼、セカンドオピニオン）
- これまでの経過と検査結果
- 紹介先への具体的なリクエスト

### 退院サマリー

退院サマリーは「入院の全体像」を簡潔にまとめる必要があります。AIは時系列整理が得意なので、経過の構造化を任せるのが効果的です。ただし、「なぜその治療方針を選んだか」という臨床判断の部分は医師自身が記載すべきです。

### 診断書・意見書

定型的な部分（病名、治療期間、経過）はAIに任せ、意見・見解の部分は医師が書く、という役割分担が適切です。法的文書であるため、最終確認は特に慎重に行いましょう。

## 品質チェックリスト

AI生成文書を確定する前に、以下を確認してください。

- [ ] 患者氏名・ID・日付は正しいか
- [ ] 病名・手術名に誤りはないか
- [ ] 薬剤名・用量は正確か（ハルシネーションの温床）
- [ ] 検査データは実際の値と一致しているか
- [ ] アレルギー情報は正しく記載されているか
- [ ] 「書くべきだが書かれていない」情報はないか
- [ ] 個人情報保護の観点で問題はないか

## 導入のコツ

1. **小さく始める**: まずは紹介状のテンプレート生成から
2. **ダブルチェックを習慣化**: AI生成→医師確認のフローを確立
3. **チーム内で共有**: 良いプロンプトはチームの共有財産にする
4. **フィードバック**: AIの出力を改善するために、修正点を記録する

AI文書作成は、医師の負担を軽減し、患者との対話時間を増やす有効な手段です。ただし、「AIが書いたから正しい」という思い込みは禁物です。最終的な文書の責任は常に医師にあることを忘れずに活用しましょう。

## なぜ抗菌薬適正使用にAIか

抗菌薬の不適切な使用は**耐性菌の増加**と**医療コストの上昇**に直結します。WHOは薬剤耐性（AMR）を「人類の健康に対する最大の脅威の一つ」と位置づけています。

研修医が抗菌薬処方で直面する課題:
- 経験的治療（empiric therapy）の選択に自信がない
- 培養結果が出た後のde-escalation（狭域化）のタイミングが分からない
- 施設のアンチバイオグラムを活用できていない
- 腎機能に応じた用量調整の計算が複雑

AIを活用することで、**エビデンスに基づいた抗菌薬選択の支援**と**de-escalationの判断補助**が可能になります。

## De-escalationのタイミングチャート

```
培養提出
  ↓
24-48時間: グラム染色結果
  → グラム陽性球菌 → バンコマイシンの適応を再評価

  ↓
72時間: 臨床効果判定
  → 改善: 治療継続、投与期間を決定
  → 不変: 検査追加（画像、再培養、感染源精査）
  → 悪化: 抗菌薬変更、外科的介入の検討、上級医相談
```

## よくある抗菌薬の間違いとAIでの予防

| 間違いパターン | AIでの予防方法 |
|---|---|
| 培養陰性でも広域抗菌薬を漫然と継続 | 「培養陰性+臨床改善→中止検討」をAIに確認 |
| 腎機能変化に対する用量未調整 | 腎機能データを入力し用量再計算 |
| ペニシリンアレルギー申告で全βラクタム回避 | 交差反応のリスク評価をAIで確認（実際の交差反応率は低い） |
| MRSAカバーの不要な継続 | 鼻腔スワブ陰性+培養MSSA→バンコマイシン中止の根拠確認 |
| 治療期間の延長しすぎ | 最新ガイドラインの推奨治療期間をAIで確認 |

## 安全に関する注意

- AIの抗菌薬推奨は**参考情報**です。処方の最終判断は必ず指導医・感染症専門医と確認してください
- 薬剤の**用量・禁忌・相互作用は添付文書やUpToDate等で必ず確認**してください
- 施設の**アンチバイオグラム**はAIよりも自施設のデータが優先されます
- 重症感染症では**感染症科コンサルト**を積極的に行ってください
- 患者データを外部AIに入力する場合は**匿名化**と施設のポリシーに準拠してください
